# 2026-06-23 — Bridge v4 (Modelo 7b + RAG + Cache)

## Decisao
Usar qwen2.5-coder:7b como modelo padrao do bridge, com fallback para 1.5b. Integrar RAG (nomic-embed-text) e cache quente para consistencia.

## Motivo
- Modelo 7b tem qualidade muito superior ao 1.5b (menos alucinacao, melhor encoding)
- RAG permite respostas contextualizadas com o codigo real do projeto
- Cache quente garante respostas consistentes para perguntas repetidas
- Fallback 1.5b evita queda total se 7b falhar (VRAM, timeout)

## Alternativas rejeitadas
- 1.5b puro: alucinava frequentemente (inventava senhas, dados)
- Apenas templates: respostas genéricas demais
- Modelo externo (API): depende de internet, maior latencia

## Referencias
- `scripts/bridge_auto.py` — implementacao do bridge v4
- `scripts/rag_query.py` — consulta ao indice RAG
- `.rag_db/` — indice vetorial com ~2300 chunks
