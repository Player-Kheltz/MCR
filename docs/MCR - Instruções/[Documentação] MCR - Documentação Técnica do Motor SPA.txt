📘 Documentação Técnica do Motor SPA – Projeto MCR (v4.1)
Última atualização: 23 de Maio de 2026

Este documento descreve todas as APIs, métodos, constantes, estruturas de dados e sistemas que o servidor Canary expõe ao Sistema de Progressão do Aventureiro. Todas as funcionalidades aqui listadas estão implementadas, compiladas e verificadas através do comando !testespa.

    50, 120, 300, 740, 1800, 2500, 3450, 4750, 6550, 9000,

1.4 Hierarquia de Domínios
A tabela DOMAIN_TREE em constantes.lua define as relações de parentesco. Os IDs primários, secundários e especialidades estão consolidados e incluem os novos domínios de Escudo (15), Runologia (22), Desarmado Puro (130), Armas de Punho (132), Bastões Arcanos (133), Arremesso de Escudo (140), etc.

Buffs temporários (geridos pelo buff_system.lua)

Speed Burst (applySpeedBurst): acelera ataque por X ms, usando sistema de token para evitar conflitos com a postura.

Damage Boost (applyDamageBoost): +% dano por N ataques.

Life Leech / Mana Leech (applyLifeLeech, applyManaLeech): roubo de vida/mana por X segundos. Processados diretamente no evento PosturaDmg via postura.lua (sem depender das skills nativas).

Next Attack Condition: aplica condição (sangramento, atordoamento, etc.) no próximo ataque.

🎯 4. Habilidades (Skills do SPA)
As habilidades são definidas em tabelas Lua (HABILIDADES[habId] = {…}) e executadas pelo executor_habilidades.lua. Cada habilidade segue o gabarito documentado em habilidade_gabarito.lua.

Tipos de habilidade

Tipo	Descrição
"passiva"	Efeito permanente (aplicado no login e ao subir de nível).
"gatilho"	Disparada por eventos (onAttack, onKill, onBlock, onReceiveDamage).
"transformacao"	Activada/desactivada com base nos Estados de Alma (Lampejo/Vínculo).
Campos principais (ver gabarito para lista completa)

nome, tipo, dominio, nivelMin

gatilho, cooldown, prioridade, categoria

buffType (speedBurst, nextAttackCondition, damageBoost, lifeLeech, custom)

efeito (função Lua), efeitoVisual, posturaMod

condicao ("cercado", "cercadoAlvo", "vidaBaixa", "distancia4", "5Golpes")

Nova condição "cercadoAlvo": verifica se o alvo tem pelo menos 2 outros inimigos num raio de 2 tiles. Ideal para habilidades de dano em área centradas no alvo (explosões, ricochetes, magias).

Execução

O executor_habilidades.lua (v6.5.1) utiliza um motor de seleção inteligente com prioridade contextual, em vez de depender de chanceBase:

Filtra habilidades elegíveis (fora de cooldown, arma compatível, condições cumpridas, Chi suficiente).

Classifica‑as por pontuação de prioridade (categoria, postura, contexto de combate).

Escolhe a melhor ou uma das 3 melhores ponderadamente.

Respeita cooldowns individuais por habilidade (campo cooldown, padrão 4s).

Suporta combos de até 3 habilidades consecutivas desbloqueados por nível de maestria (chances configuráveis).

Aplica um cooldown global entre sequências que diminui com o nível de maestria (6.0s → 3.0s).

🔫 5.1 Sistema de Projéteis Dinâmico
A função global getProjectile(player) em util_armas.lua (v6.1) resolve automaticamente o projétil visual correto:

Armas de distância (arcos/bestas): usa player:getAmmo() (novo método C++) para obter a munição real que está dentro da aljava ou slot de munição, e devolve o shootType dessa munição.

Armas de arremesso (MISSILE): usa item:getShootType() da própria arma.

Wands: usa item:getShootType() da wand.

Fallback genérico por tipo de arma caso nenhum método retorne um valor válido.

Isto garante que habilidades de múltiplos projéteis exibam sempre a animação correta (flecha flamejante, power bolt, estrela de arremesso, etc.), sem necessidade de tabelas manuais.

🌐 6. Barramento de Eventos Universal (Event Bus)
Implementado via SPAManager (C++) e exposto como SPA.onEvent / SPA.dispatchEvent.

Métodos Lua

Método Lua	Descrição
SPA.onEvent(eventName, callback)	Regista uma função Lua para ser chamada quando o evento ocorrer.
SPA.dispatchEvent(eventName, …)	Dispara um evento, chamando todos os callbacks registados.
Eventos já disponíveis (disparados pelo motor)

Evento	Argumentos	Local de disparo
"onPlayerDealtDamage"	player, target, damage, primaryType, secondaryType	combat.cpp
"onPlayerKill"	player, target	eventos.lua (via ProgKill)
"onPlayerLogin"	player	player.cpp (onLogin)
Novos eventos podem ser adicionados facilmente chamando SPA.dispatchEvent em qualquer script Lua ou ponto do C++.

⚙️ 11. Performance e Modularidade
Todas as consultas de afinidade em lote usam addDominioAfinidadeBatch.

O buffer de progressão descarrega a cada 2 segundos, minimizando escritas na BD.

Os eventos Lua são registados dinamicamente; novos domínios e habilidades podem ser adicionados sem recompilar.

A tabela SPA é global; qualquer script pode estender o sistema.

Os thinks do jogador foram unificados no think_master.lua (redução de 4 eventos para 1).

O sistema de token para Speed Burst evita conflitos entre buff_system.lua e postura.lua, sem depender de os.mtime().
