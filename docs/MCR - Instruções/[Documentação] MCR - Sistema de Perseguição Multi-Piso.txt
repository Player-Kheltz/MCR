1. VISÃO GERAL
O sistema de perseguição multi‑piso permite que criaturas (summons do jogador e monstros hostis) persigam um alvo através de múltiplos andares.
A base do sistema é uma fila de transições de escada (StairTransition) que o jogador vai deixando ao mudar de piso. As criaturas consultam essa fila, encontram a transição adequada ao seu piso atual e caminham até ao ponto de origem da escada. Ao chegar, utilizam a escada (por movimento natural ou teleporte forçado para escadas ativas) e continuam a perseguição no piso seguinte.

Além das transições de piso, o jogador também pode largar waypoints de superfície (“migalhas”) no mesmo andar, permitindo que criaturas bloqueadas contornem obstáculos seguindo o rasto exato do jogador.

1.1 Tipos de Criaturas e Comportamentos
Tipo	Perseguição	Timeout	Retorno	Uso de Migalhas
Summon	Segue o mestre através de escadas	120 ciclos (~120s)	Teleporta para o mestre após timeout	Sim, se o mestre estiver inacessível
Hostil	Persegue o jogador que está a atacar	45 segundos reais	Regressa a pé ao spawn (ou à posição inicial)	Sim, quando bloqueado a caminho da escada

3. FLUXO PRINCIPAL DE PERSEGUIÇÃO
3.1 onThink_async (monster.cpp)
A cada ciclo de pensamento, o monstro verifica se deve iniciar ou continuar uma perseguição.

cpp
// Verificar se deve começar a perseguir
if (isSummon() || (!isSummon() && (!m_attackedCreature.expired() ||
    (m_persistentTargetPlayer && !m_persistentTargetPlayer->getStairTransitions().empty())))) {
    checkStairPursuit();
}
Condições para iniciar:

Summons: sempre verificam.

Hostis: verificam se têm um alvo de ataque ativo ou se existe um jogador memorizado com transições pendentes.

3.2 checkStairPursuit (monster.cpp)
Função que determina qual a próxima transição a perseguir e inicia a caminhada.

Passos:

Obtém o líder (mestre para summons, alvo de ataque ou jogador memorizado para hostis).

Guarda o Player do líder (para hostis, atualiza m_persistentTargetPlayer).

Se a fila de transições do jogador estiver vazia:

Summon: se estiver num piso diferente do mestre, teleporta.

Summon (mesmo piso, inacessível): tenta seguir waypoints de superfície.

Hostil: simplesmente aguarda.

Inicializa m_nextStairIndex (0 para hostis, fila.size() para novos summons).

Percorre a fila a partir do índice guardado, procurando uma transição cuja origem esteja no seu piso atual.

Ao encontrar, inicia a perseguição (m_pursuingStairs = true), guarda os dados da transição e chama walkToWaypoint.

3.3 Bloco de Perseguição no onThink_async
Quando m_pursuingStairs == true, o monstro:

Atualiza contadores e verifica timeouts.

Verifica se o líder ficou acessível (apenas para summons; para hostis, mantém-se focado na escada).

Verifica se chegou ao waypoint (escada ou waypoint de superfície).

Monitoriza caminhadas bloqueadas e aplica desencravamento.

Tenta novos caminhos se necessário.

4. MECANISMOS DE TIMEOUT E DESISTÊNCIA
4.1 Timeout Total (Hostis)
Os hostis têm 45 segundos reais (getCurrentTimeMs() - m_pursuitStartTime > 45000) para completar a perseguição.
Se o tempo expirar sem alcançar o alvo:

Se tiverem usado escadas (!m_returnPath.empty()), iniciam o retorno ao spawn.

Caso contrário, regressam a pé à posição onde a perseguição começou (m_homePosition).

4.2 Timeout de Progresso (Hostis)
Se um hostil ficar 4 segundos sem reduzir a distância ao waypoint atual (e não estiver a caminhar), abandona essa escada e avança para a próxima transição (m_nextStairIndex++).

Log de exemplo:

text
[MCR-STAIR] Rato bloqueado há 4s sem progresso, abandonando esta escada.
4.3 Falhas de Movimento Forçado (Hostis)
Se um hostil tenta forçar um passo (em linha reta) e a posição não muda 3 vezes consecutivas, abandona a escada.

Log de exemplo:

text
[MCR-STAIR] Rato movimento forçado falhou (falha 1).
[MCR-STAIR] Rato movimento forçado falhou (falha 2).
[MCR-STAIR] Rato movimento forçado falhou (falha 3).
[MCR-STAIR] Rato 3 falhas de movimento forçado, abandonando esta escada.
4.4 Timeout de Summons
Summons têm 120 ciclos de onThink para alcançar a escada; se excederem, teleportam diretamente para o destino.

5. WAYPOINTS DE SUPERFÍCIE (MIGALHAS)
Para ajudar criaturas bloqueadas no mesmo piso, o jogador vai largando pontos de passagem (SurfaceWaypoint) ao caminhar.

5.1 Geração de Waypoints
Método: Player::addSurfaceWaypoint(getPosition())
Chamado em: Player::onWalkComplete (ou equivalente)

Regras:

Só adiciona se a distância Manhattan para o último waypoint for ≥ 5 tiles.

Remove waypoints expirados (mais de 2 minutos sem uso).

A lista de waypoints é partilhada e cada criatura tem o seu índice pessoal.

Log de exemplo:

text
[MCR-PLAYER] Kheltz novo waypoint de superfície adicionado. Total: 3.
5.2 Uso por Summons
Quando um summon está no mesmo piso que o mestre, mas inacessível (pathfinding falha), ele tenta seguir os waypoints de superfície do mestre.

Log de exemplo:

text
[MCR-STAIR] CriaturaSPA a seguir waypoint de superfície #2 em (1019,1008,6).
5.3 Uso por Hostis
Quando um hostil fica bloqueado a caminho de uma escada (stuckCount >= 8), ele procura o waypoint de superfície do jogador mais próximo da escada e caminha até lá.

Log de exemplo:

text
[MCR-STAIR] Rato preso, a seguir waypoint de superfície para a escada (1019,1008,6).
5.4 Atualização de Timestamp
Quando uma criatura alcança um waypoint, o timestamp é atualizado (touchSurfaceWaypoint), prolongando a sua vida por mais 2 minutos. Isto garante que múltiplos summons em fila não percam as referências.

6. RETORNO AO SPAWN / POSIÇÃO INICIAL
6.1 Retorno com Escadas (iniciarRetornoSpawn)
Quando um hostil usou escadas durante a perseguição e atinge o timeout de 45s, ele inicia o retorno ao spawn percorrendo o m_returnPath ao contrário.

Processo:

Inverte m_returnPath.

Para cada transição, caminha até ao destino original e teleporta de volta pela escada.

Só usa teleporte de emergência se estiver completamente bloqueado e não houver jogadores num raio de 8 tiles.

Log de exemplo:

text
[MCR-STAIR] Rato iniciou retorno ao spawn com 3 transições.
[MCR-STAIR] Rato a caminhar de volta para (1019,1014,6).
[MCR-STAIR] Rato usando escada de retorno para (1018,1015,7).
[MCR-STAIR] Rato retorno ao spawn concluído.
6.2 Retorno Simples (processarRetornoSimples)
Quando o hostil não usou escadas, ele regressa a pé até m_homePosition (posição onde a perseguição começou).

Log de exemplo:

text
[MCR-STAIR] Rato sem escadas percorridas, voltando a pé para casa (1015,1015,7).
[MCR-STAIR] Rato chegou a casa.

8. INTEGRAÇÃO COM SUMMONS VS HOSTIS
Funcionalidade	Summons	Hostis
Iniciar perseguição	Sempre que o mestre muda de piso	Quando têm target e o alvo muda de piso
Cancelar perseguição	Se o mestre fica acessível	Apenas se o líder estiver acessível e a ≤ 3 tiles
Timeout	120 ciclos (~120s)	45 segundos reais
Desistência por bloqueio	Teleporta após 5 falhas de pathfinding	Abandona a escada após 4s sem progresso ou 3 movimentos forçados falhados
Retorno	Teleporta para o mestre	Caminha de volta ao spawn ou à posição inicial
Waypoints de superfície	Usa se o mestre estiver inacessível	Usa quando bloqueado a caminho da escada
Aguardar aliados	Sim (fila indiana)	Não

10. EXEMPLOS DE COMPORTAMENTO
10.1 Hostil Segue o Jogador por Várias Escadas
Jogador sobe uma escada → transição adicionada.

Rato inicia perseguição, caminha até à escada, muda de piso.

Jogador usa outra escada → nova transição adicionada.

Rato processa a próxima transição, sobe a segunda escada.

Continua até alcançar o jogador ou atingir o timeout.

10.2 Hostil Bloqueado Usa Migalhas
Jogador caminha por um corredor com curva, largando waypoints a cada 5 tiles.

Jogador usa uma escada no final do corredor.

Rato tenta seguir diretamente para a escada, mas o pathfinding falha (parede no caminho).

Após 8 ciclos preso, o Rato procura o waypoint mais próximo da escada.

Rato caminha até ao waypoint (seguindo as migalhas), contornando a parede.

Chega à escada e muda de piso.

10.3 Hostil Regressa ao Spawn Após Timeout
Rato persegue o jogador durante 45 segundos, usando 2 escadas.

Timeout de 45s expira.

Rato inicia iniciarRetornoSpawn com 2 transições no m_returnPath.

Rato volta pela segunda escada, depois pela primeira, chegando ao piso inicial.

Retorno concluído; Rato retoma comportamento normal.

11. MANUTENÇÃO E EXPANSÃO FUTURA
11.1 Adicionar Novos Mecanismos de Desistência
Para ajustar os tempos de timeout ou o número de falhas permitidas, modifique as constantes no monster.cpp:

m_pursuitTimeout (padrão: 45 segundos)

m_forcedMoveFailCount limite (padrão: 3)

Tempo de progresso (padrão: 4000 ms)

11.2 Expor Mais Dados ao Lua
Os métodos getSurfaceWaypoints, touchSurfaceWaypoint e getStairTransitions devem ser registados em SPAFunctions::init para uso em scripts Lua (comandos de diagnóstico, etc.).

11.3 Limpeza de Dados
A fila de transições do jogador (m_stairTransitions) pode crescer indefinidamente se não for limpa. Atualmente, transições expiradas (>30s) são removidas no início de checkStairPursuit.
Os waypoints de superfície expiram 2 minutos após o último uso.

void Player::onFloorChange(const Position& fromPos, const Position& toPos) {
    StairTransition trans;
    trans.origin = fromPos;
    trans.destination = toPos;
    trans.timestamp = getCurrentTimeMs();
    trans.isActive = isActiveStair(fromPos);
    m_stairTransitions.push_back(trans);

    // Log
    g_logger().info("[MCR-PLAYER] {} nova transição adicionada. Fila tem {} elemento(s). origin={} dest={} isActive={}",
        getName(), m_stairTransitions.size(), trans.origin.toString(), trans.destination.toString(), trans.isActive);
}

void Player::addSurfaceWaypoint(const Position& pos) {
    uint64_t now = getCurrentTimeMs();
    // Remove waypoints expirados (último acesso > 2 minutos)
    while (!m_surfaceWaypoints.empty() && now - m_surfaceWaypoints.front().timestamp > SURFACE_WAYPOINT_EXPIRY_MS) {
        m_surfaceWaypoints.pop_front();
    }

    // Só adiciona se a distância para o último waypoint for >= 5
    if (!m_surfaceWaypoints.empty()) {
        const Position& last = m_surfaceWaypoints.back().pos;
        if (std::abs(pos.x - last.x) + std::abs(pos.y - last.y) < 5) {
            return;
        }
    }

    m_surfaceWaypoints.push_back({pos, now});
}

void Player::touchSurfaceWaypoint(size_t index) {
    if (index < m_surfaceWaypoints.size()) {
        m_surfaceWaypoints[index].timestamp = getCurrentTimeMs();
    }
    // Remove imediatamente waypoints expirados da frente
    uint64_t now = getCurrentTimeMs();
    while (!m_surfaceWaypoints.empty() && now - m_surfaceWaypoints.front().timestamp > SURFACE_WAYPOINT_EXPIRY_MS) {
        m_surfaceWaypoints.pop_front();
    }
}

4.1 checkStairPursuit
cpp
void Monster::checkStairPursuit() {
    g_logger().info("[MCR-STAIR] {}::checkStairPursuit() chamado. pursuingStairs={}", getName(), m_pursuingStairs);

    if (m_pursuingStairs || m_returningToSpawn) {
        g_logger().info("[MCR-STAIR] {} já está em perseguição ou retorno, a ignorar.", getName());
        return;
    }

    std::shared_ptr<Creature> leader;
    if (isSummon()) {
        leader = getMaster();
    } else {
        if (!m_attackedCreature.expired()) {
            leader = m_attackedCreature.lock();
        } else if (m_persistentTargetPlayer) {
            leader = m_persistentTargetPlayer;
        }
    }

    if (!leader) {
        g_logger().info("[MCR-STAIR] {} sem líder, a abortar.", getName());
        return;
    }

    auto leaderPlayer = leader->getPlayer();
    if (!leaderPlayer) {
        g_logger().info("[MCR-STAIR] {} líder não é jogador, a abortar.", getName());
        return;
    }

    if (!isSummon()) {
        if (m_persistentTargetPlayer != leaderPlayer) {
            m_nextStairIndex = 0;
        }
        m_persistentTargetPlayer = leaderPlayer;
    }

    auto& fila = leaderPlayer->getStairTransitions();

    if (fila.empty()) {
        if (isSummon()) {
            if (leader->getPosition().z != getPosition().z) {
                g_logger().info("[MCR-STAIR] {} fila vazia e piso diferente do mestre. Teleportando para mestre.", getName());
                g_game().internalTeleport(static_self_cast<Monster>(), leader->getPosition());
            } else {
                FindPathParams fpp;
                getPathSearchParams(getMonster(), fpp);
                std::vector<Direction> dummy;
                if (!getPathTo(leader->getPosition(), dummy, fpp)) {
                    auto& waypoints = leaderPlayer->getSurfaceWaypoints();
                    if (m_currentSurfaceWaypointIndex >= waypoints.size()) {
                        m_currentSurfaceWaypointIndex = 0;
                    }
                    while (m_currentSurfaceWaypointIndex < waypoints.size()) {

            m_stairAttemptStartTime = getCurrentTimeMs();

            m_lastDistImprovementTime = getCurrentTimeMs();

                getName(), m_stairOrigin.toString(), m_stairDestination.toString(), m_stairIsActive, m_savedFollowId);

        if (leader->getPosition().z != getPosition().z) {

            m_nextStairIndex = fila.size();
        }

        g_logger().info("[MCR-STAIR] {} hostil sem transições compatíveis, a aguardar.", getName());
        if (m_attackedCreature.expired() && m_persistentTargetPlayer &&
            m_persistentTargetPlayer->getStairTransitions().empty()) {
            m_persistentTargetPlayer.reset();
        }
    }

                    m_nextStairIndex = m_currentTransitionIndex + 1;
                    clearStairPursuit();

                return;
            }
            return;
        }

        if (isSummon() && dist > 0 && m_stairStuckCount < 3) {
            Direction dirToWaypoint = getDirectionTo(myPos, m_stairOrigin);

void Monster::walkToWaypoint(const Position& target) {
    g_logger().info("[MCR-WALK] {}::walkToWaypoint de {} para {}", getName(), getPosition().toString(), target.toString());

    std::vector<Direction> listDir;
    FindPathParams fpp;
    getPathSearchParams(getMonster(), fpp);
    fpp.minTargetDist = 0;
    fpp.maxTargetDist = 1;
    fpp.fullPathSearch = true;
    fpp.clearSight = false;
    fpp.maxSearchDist = 0;

    if (getPathTo(target, listDir, fpp) && !listDir.empty()) {
        g_logger().info("[MCR-WALK] {} caminho encontrado: {} passos. Primeiro passo: {}", getName(), listDir.size(), static_cast<int>(listDir.front()));

        setWalkingToPosition(true);

            m_minDistToStair = std::abs(getPosition().x - target.x) + std::abs(getPosition().y - target.y);

        Direction dir = getDirectionTo(getPosition(), target);
        if (dir != DIRECTION_NONE) {
            g_logger().info("[MCR-WALK] {} direcção forçada: {}", getName(), static_cast<int>(dir));
            g_game().internalMoveCreature(static_self_cast<Monster>(), dir, 0);

        }
    }

void Monster::iniciarRetornoSpawn() {
    clearStairPursuit();
    m_persistentTargetPlayer.reset();

    if (m_returnPath.empty()) {
        g_logger().info("[MCR-STAIR] {} sem caminho de retorno, a voltar ao comportamento normal.", getName());
        return;
    }

    std::reverse(m_returnPath.begin(), m_returnPath.end());
    m_returningToSpawn = true;
    g_logger().info("[MCR-STAIR] {} iniciou retorno ao spawn com {} transições.", getName(), m_returnPath.size());
    processarRetornoSpawn();
}

void Monster::processarRetornoSpawn() {
    if (!m_returningToSpawn || m_returnPath.empty()) {
        m_returningToSpawn = false;
        m_walkingForReturn = false;
        m_nextStairIndex = 0;
        clearStairPursuit();
        g_logger().info("[MCR-STAIR] {} retorno ao spawn concluído.", getName());
        return;
    }

    if (isWalkingToPosition()) return;

    StairTransition& trans = m_returnPath.back();
    Position myPos = getPosition();

    if (myPos == trans.destination) {
        g_logger().info("[MCR-STAIR] {} usando escada de retorno para {}.", getName(), trans.origin.toString());
        g_game().internalTeleport(static_self_cast<Monster>(), trans.origin);
        m_returnPath.pop_back();
        if (m_returnPath.empty()) {
            m_returningToSpawn = false;
            m_walkingForReturn = false;
            m_nextStairIndex = 0;
            clearStairPursuit();
            g_logger().info("[MCR-STAIR] {} retorno ao spawn concluído.", getName());
        } else {
            processarRetornoSpawn();
        }
        return;
    }

    g_logger().info("[MCR-STAIR] {} a caminhar de volta para {}.", getName(), trans.destination.toString());
    walkToWaypoint(trans.destination);
    m_walkingForReturn = true;

    if (!isWalkingToPosition()) {
        Direction dir = getDirectionTo(myPos, trans.destination);
        if (dir != DIRECTION_NONE) {
            g_logger().info("[MCR-STAIR] {} sem caminho, movimento simples na direção do retorno.", getName());
            g_game().internalMoveCreature(static_self_cast<Monster>(), dir, 0);
            m_walkingForReturn = true;
        } else {
            if (!isPlayerNearby(8)) {
                g_logger().info("[MCR-STAIR] {} sem caminho e sem jogadores, teleportando para origem {}.", getName(), trans.origin.toString());
                g_game().internalTeleport(static_self_cast<Monster>(), trans.origin);
                m_returnPath.pop_back();
                processarRetornoSpawn();
            } else {
                g_logger().info("[MCR-STAIR] {} não pode teleportar (jogadores próximos), aguardando.", getName());
                m_walkingForReturn = false;
            }
        }
    }
}

void Monster::processarRetornoSimples() {
    if (!m_returningHome) return;
    if (isWalkingToPosition()) return;

    Position myPos = getPosition();
    if (myPos == m_homePosition || (std::abs(myPos.x - m_homePosition.x) + std::abs(myPos.y - m_homePosition.y) <= 1)) {
        g_logger().info("[MCR-STAIR] {} chegou a casa.", getName());
        m_returningHome = false;
        m_walkingForReturn = false;
        return;
    }

    g_logger().info("[MCR-STAIR] {} a caminhar para casa {}.", getName(), m_homePosition.toString());
    walkToWaypoint(m_homePosition);
    m_walkingForReturn = true;

    if (!isWalkingToPosition()) {
        Direction dir = getDirectionTo(myPos, m_homePosition);
        if (dir != DIRECTION_NONE) {
            g_logger().info("[MCR-STAIR] {} sem caminho, movimento simples na direção de casa.", getName());
            g_game().internalMoveCreature(static_self_cast<Monster>(), dir, 0);
            m_walkingForReturn = true;
        } else {
            g_logger().info("[MCR-STAIR] {} completamente bloqueado, a desistir do retorno.", getName());
            m_returningHome = false;
            m_walkingForReturn = false;
        }
    }
}

                    if tile:getTopCreature() then ch = "C"
                    elseif tile:hasFlag(TILESTATE_FLOORCHANGE) then ch = "S"
                    elseif tile:hasFlag(TILESTATE_PROTECTIONZONE) then ch = "P"

            for ch in l:gmatch(".") do

                elseif ch == "C" then lc = lc .. c("C", COR.ITEM_RARO)

                else lc = lc .. c(".", COR.SISTEMA_NEUTRO) end

            enviarMsgColorida(player, lc)
        end
    end
    return false
end

function talkScanFull.onSay(player, words, param)
    local params = param:split(" ")
    local raio = tonumber(params[1]) or 5
    local zRange = tonumber(params[2]) or 2
    if raio > 10 then enviarMsgColorida(player, c("Raio máximo 10.", COR.SISTEMA_ERRO)) return false end
    if zRange > 3 then enviarMsgColorida(player, c("Z-Range máximo 3.", COR.SISTEMA_ERRO)) return false end

    local pos = player:getPosition()
    local filename = string.format("data/logs/scan/scan_%s_%s.log", os.date("%Y%m%d_%H%M%S"), player:getName():gsub("[^%w]", "_"))
    os.execute("mkdir \"data/logs/scan\" 2> NUL")
    local file = io.open(filename, "w")
    if not file then
        enviarMsgColorida(player, c("Erro ao criar arquivo.", COR.SISTEMA_ERRO))
        return false
    end

    file:write("=== SCAN ULTRA DETALHADO ===\n")
    file:write(string.format("Jogador: %s (ID:%d) Pos: %d,%d,%d Raio: %d Z-Range: %d\n\n", player:getName(), player:getId(), pos.x, pos.y, pos.z, raio, zRange))

    -- Waypoints de superfície
    local ok, wps = pcall(player.getSurfaceWaypoints, player)
    file:write("=== WAYPOINTS DE SUPERFÍCIE ===\n")
    if ok and wps then
        file:write(string.format("Total: %d\n", #wps))
        for i, wp in ipairs(wps) do
            file:write(string.format("  #%d: %d,%d,%d (timestamp:%d)\n", i-1, wp.x or 0, wp.y or 0, wp.z or 0, wp.timestamp or 0))
        end
    else file:write("Não disponível.\n") end
    file:write("\n")

    -- Transições de escada
    local ok2, trans = pcall(player.getStairTransitions, player)
    file:write("=== TRANSIÇÕES DE ESCADA ===\n")
    if ok2 and trans then
        file:write(string.format("Total: %d\n", #trans))
        for i, t in ipairs(trans) do
            file:write(string.format("  #%d: Origem:%d,%d,%d -> Destino:%d,%d,%d | Active:%s\n",
                i-1, t.originX or 0, t.originY or 0, t.originZ or 0, t.destX or 0, t.destY or 0, t.destZ or 0, t.isActive and "SIM" or "NÃO"))
        end
    else file:write("Não disponível.\n") end
    file:write("\n")

    for z = pos.z + zRange, pos.z - zRange, -1 do
        file:write(string.format("=== ANDAR Z=%d ===\n", z))
        for y = pos.y - raio, pos.y + raio do
            for x = pos.x - raio, pos.x + raio do
                local tile = Tile(Position(x, y, z))
                if tile then
                    file:write(string.format("\n>>> TILE (%d,%d,%d) GroundID:%d\n", x, y, z, tile:getGround() and tile:getGround():getId() or 0))
                    local flags = {}
                    if tile:hasFlag(TILESTATE_BLOCKPATH) then table.insert(flags, "BLOCKPATH") end
                    if tile:hasFlag(TILESTATE_FLOORCHANGE) then table.insert(flags, "FLOORCHANGE") end
                    file:write(string.format("    Flags: %s\n", table.concat(flags, " | ")))
                    local items = tile:getItems()
                    file:write(string.format("    Itens: %d\n", #items))
                    for _, item in ipairs(items) do
                        file:write(string.format("      ID:%d Nome:'%s'\n", item:getId(), item:getName() or "?"))
                    end
                    local creature = tile:getTopCreature()
                    if creature then
                        file:write(string.format("    Criatura: %s (ID:%d)\n", creature:getName() or "?", creature:getId()))
                    end
                else
                    file:write(string.format("\n>>> TILE (%d,%d,%d) NULO\n", x, y, z))
                end
            end
        end
    end

    file:write("\n=== FIM DO SCAN ===\n")
    file:close()
    enviarMsgColorida(player, c("Scan ultra detalhado salvo em: " .. filename, COR.SUCESSO_CURA))
    return false
end

// Em spa_functions.cpp
#include "creatures/players/player.hpp"

int SPAFunctions::luaPlayerGetSurfaceWaypoints(lua_State* L) {
    auto player = Lua::getUserdataShared<Player>(L, 1, "Player");
    if (!player) { lua_newtable(L); return 1; }
    const auto& waypoints = player->getSurfaceWaypoints();
    lua_newtable(L);
    int index = 1;
    for (const auto& wp : waypoints) {
        lua_newtable(L);
        Lua::setField(L, "x", wp.pos.x);
        Lua::setField(L, "y", wp.pos.y);
        Lua::setField(L, "z", wp.pos.z);
        Lua::setField(L, "timestamp", static_cast<lua_Number>(wp.timestamp));
        lua_rawseti(L, -2, index++);
    }
    return 1;
}

int SPAFunctions::luaPlayerTouchSurfaceWaypoint(lua_State* L) {
    auto player = Lua::getUserdataShared<Player>(L, 1, "Player");
    if (!player) { lua_pushnil(L); return 1; }
    uint32_t index = Lua::getNumber<uint32_t>(L, 2);
    player->touchSurfaceWaypoint(index);
    lua_pushboolean(L, true);
    return 1;
}

int SPAFunctions::luaPlayerGetStairTransitions(lua_State* L) {
    auto player = Lua::getUserdataShared<Player>(L, 1, "Player");
    if (!player) { lua_newtable(L); return 1; }
    const auto& transitions = player->getStairTransitions();
    lua_newtable(L);
    int index = 1;
    for (const auto& trans : transitions) {
        lua_newtable(L);
        Lua::setField(L, "originX", trans.origin.x);
        Lua::setField(L, "originY", trans.origin.y);
        Lua::setField(L, "originZ", trans.origin.z);
        Lua::setField(L, "destX", trans.destination.x);
        Lua::setField(L, "destY", trans.destination.y);
        Lua::setField(L, "destZ", trans.destination.z);
        Lua::setField(L, "isActive", trans.isActive);
        Lua::setField(L, "timestamp", static_cast<lua_Number>(trans.timestamp));
        lua_rawseti(L, -2, index++);
    }
    return 1;
}
