Ao criar ou modificar QUALQUER script de NPC (.lua) no servidor, o assistente DEVE:

- Todo texto exibido ao jogador em PT‑BR impecável, com acentuação correcta. O arquivo DEVE ser salvo como ISO‑8859‑1 (Latin‑1). Strings com acentos salvos em UTF‑8 PROVOCAM CORRUPÇÃO no cliente.

  - Criar o tipo com `Game.createNpcType(npcName)`
  - Atribuir todos os callbacks de engine (onThink, onAppear, onDisappear, onMove, onSay, onCloseChannel) delegando ao `npcHandler`.
  - Registrar o NPC com `npcType:register(npcConfig)`.
  - Usar a assinatura correcta do `creatureSayCallback`: `(npc, player, type, msg)`.
- `npcHandler:say` aceita string única ou tabela de strings com delay.
- Usar o módulo `pronouns.lua` para obter pronomes do personagem:

  - Usar `Pronouns.getSubject(player)`, `Pronouns.getObject(player)`, etc.
  - NÃO usar storage 100001 para pronome do personagem.
- A coluna nativa `players.pronoun` é preenchida pelo Oráculo e usada automaticamente pelo módulo.
- Respeitar a distinção entre:
  - **Pronome da CONTA** (jogador/jogadora): usado apenas pelo Oráculo.
  - **Pronome do PERSONAGEM** (ele/ela): usado por todos os NPCs e mensagens.
- Palavras importantes no chat DEVEM ser destacadas com `{chaves}`, que o OTClient Redemption exibe como destaque visual automático. Adicionalmente, usar `c()` para colorir. - Diálogos NÃO devem usar `addEvent` manual; para sequências de falas, utilize a capacidade do handler de receber uma tabela de strings.
- Toda interação administrativa (loja, banco) pode usar janela modal, mas DEVE ser precedida de uma fala narrativa no chat.
- Incluir prints de debug (`"DEBUG:"`) em todos os callbacks do NPC, especialmente em `creatureSayCallback`, `greetCallback` e `farewellCallback`, documentando a entrada de cada bloco condicional, mudanças de tópico e decisões tomadas.

- A forma como o NPC pergunta ou infere o pronome do personagem DEVE ser coerente com a personalidade do NPC.
- A progressão de intimidade com o NPC deve refletir‑se na linguagem.
- Para tradução de diálogos existentes, a pipeline de tradução automática pode ser usada. Após a tradução, os ficheiros Lua devem ser verificados e salvos como ISO‑8859‑1.

- ⚠️ SPA v4.2: O NPC pode comentar Lampejos, Vínculos e a Maestria de forma subtil, usando `player:getDominioState(dominioId)` e `player:getDominioAfinidade(dominioId)`.

🌌 1. TOM DE VOZ E AMBIENTAÇÃO – O UNIVERSO MCR

O Projeto MCR se passa em um universo de fantasia original. NPCs NÃO são terminais de missão, mas habitantes vivos. Ao criar um NPC, reflita:

- Qual a história dele dentro do mundo?
- Como ele reage a guerreiros, magos, druidas? (Nota: no MCR não há vocações, mas o NPC pode reconhecer um Aventureiro com forte afinidade marcial, mágica, elemental, etc.)
- Qual seu medo, objetivo, mania?
- Como a sua personalidade influencia a maneira como se dirige ao jogador? (ex.: um ferreiro rude fala de forma brusca; uma sacerdotisa é calorosa)
- Como ele percebe o crescimento do Aventureiro? Um mestre espadachim pode notar a evolução nos Domínios de Combate; um arquivista pode sentir a aura mágica de um Vínculo elemental; um mago ancião pode reconhecer a Sinergia Elemental quando o Aventureiro empunha uma arma imbuída de fogo.
- Como reage ao ver um Aventureiro usando apenas um escudo, arremessando‑o como arma?

📚 2. MÓDULO PRONOUNS.LUA (OBRIGATÓRIO)

Arquivo: `data-canary/scripts/MCR/Geral/pronouns.lua`
Salvar como: ISO‑8859‑1

Este módulo encapsula o sistema nativo de pronomes do Canary e é carregado automaticamente pelo Revscript. NÃO é necessário usar `dofile` para carregá-lo. Ele oferece funções simples que automaticamente consultam a coluna `players.pronoun` e fazem fallback para o sexo do personagem quando o pronome não está definido.

Funções disponíveis:

- `Pronouns.getSubject(player)` → `"he"`, `"she"`, `"they"`
- `Pronouns.getObject(player)` → `"him"`, `"her"`, `"them"`
- `Pronouns.getPossessive(player)` → `"his"`, `"her"`, `"their"`
- `Pronouns.getReflexive(player)` → `"himself"`, `"herself"`, `"themself"`

Exemplo de uso:
```lua
local subject = Pronouns.getSubject(player)
npcHandler:say("Que " .. subject .. " seja bem‑vindo" .. (subject == "she" and "a" or "") .. "!", npc, player)
O módulo utiliza as constantes PLAYERPRONOUN_HE, PLAYERPRONOUN_SHE, etc., que são expostas pelo servidor.

🏗️ 4. ESQUELETO FUNCIONAL DE NPC COM SISTEMA DE CORES

Todo script de NPC deve seguir o modelo abaixo, agora com integração das cores do SPA v4.2. Exemplo de um ferreiro que reconhece o nível de Lâminas, a Sinergia Elemental e o Arremesso de Escudo:

lua
--[[
Projeto: MCR
NPC: Ferreiro de Eridanus (exemplo)
Descrição: Demonstração com cores v9.1 e reacção a Domínios, Sinergia Elemental e Arremesso de Escudo.
--]]

local npcName = "Ferreiro"
local npcType = Game.createNpcType(npcName)
local npcConfig = {}
npcConfig.name = npcName
npcConfig.outfit = { lookType = 128, lookHead = 0, lookBody = 0, lookLegs = 0, lookFeet = 0, lookAddons = 2 }

local keywordHandler = KeywordHandler:new()
local npcHandler = NpcHandler:new(keywordHandler)

npcType.onThink = function(npc, interval) npcHandler:onThink(npc, interval) end
npcType.onAppear = function(npc, creature) npcHandler:onAppear(npc, creature) end
npcType.onDisappear = function(npc, creature) npcHandler:onDisappear(npc, creature) end
npcType.onMove = function(npc, creature, from, to) npcHandler:onMove(npc, creature, from, to) end
npcType.onSay = function(npc, creature, type, msg) npcHandler:onSay(npc, creature, type, msg) end
npcType.onCloseChannel = function(npc, player) npcHandler:onCloseChannel(npc, player) end

local function greetCallback(npc, player)
    local subject = Pronouns.getSubject(player)
    local frase = (subject == "she") and "Bem‑vinda, nobre senhora!" or "Bem‑vindo, nobre senhor!"
    npcHandler:setMessage(MESSAGE_GREET, frase)
    return true
end
npcHandler:setCallback(CALLBACK_GREET, greetCallback)

local function creatureSayCallback(npc, player, type, msg)
    if not npcHandler:checkInteraction(npc, player) then return false end
    local subj = Pronouns.getSubject(player)

    -- Verifica nível de Lâminas (Domínio 10)
    local afinidadeLaminas = player:getDominioAfinidade(10) or 0
    local nivelLaminas = getNivelPorAfinidade(afinidadeLaminas)

    if NpcUtils.correspondeAcao(msg, {"missão", "ajuda", "tarefa", "trabalho", "aventura", "desafio"}) then
        if nivelLaminas >= 20 then
            npcHandler:say("Ah! Um " .. c("Mestre das Lâminas", COR.RARIDADE_LENDARIO) .. "! O aço reconhece‑vos. Talvez queira forjar uma " .. c("{Lâmina Rúnica}", COR.ITEM_RARO) .. "?", npc, player)
        elseif nivelLaminas >= 15 then
            npcHandler:say("Suas mãos já falam com o aço. Talvez queira forjar uma " .. c("{Lâmina Rúnica}", COR.ITEM_RARO) .. "?", npc, player)
        elseif nivelLaminas >= 5 then
            npcHandler:say("Vejo que já tem prática com a " .. c("{espada}", COR.DOM_COMBATE_LAMINAS) .. ". Precisa de algo?", npc, player)
        else
            npcHandler:say("Preciso de alguém que saiba manejar uma " .. c("{espada}", COR.DOM_COMBATE_LAMINAS) .. ". Você " .. c("{aceita}", COR.NPC_INTENCAO) .. " treinar?", npc, player)
        end
    end

    -- Comenta Maestria, Vínculo ou Lampejo
    local state = player:getDominioState(10) or 0
    local nivel = getNivelPorAfinidade(afinidadeLaminas)
    if nivel >= 20 then
        npcHandler:say("É uma honra ter um " .. c("Mestre das Lâminas", COR.RARIDADE_LENDARIO) .. " na minha forja. O aço nunca o esquecerá.", npc, player)
    elseif state == 2 then
        npcHandler:say("Sinto o Vínculo com as lâminas... o aço estremece à sua passagem.", npc, player)
    elseif state == 1 then
        npcHandler:say("Você está em Lampejo com as espadas? O som do metal é mais puro.", npc, player)
    end

    -- Comenta Sinergia Elemental se o jogador empunha uma arma de fogo
    local weapon = player:getSlotItem(CONST_SLOT_LEFT) or player:getSlotItem(CONST_SLOT_RIGHT)
    if weapon then
        local elemento = nil
        if weapon.getElementalBond then elemento = weapon:getElementalBond() end
        if not elemento and weapon.getElementType then elemento = weapon:getElementType() end
        if elemento == COMBAT_FIREDAMAGE then
            npcHandler:say("Essa lâmina carrega o " .. c("fogo", COR.ELEM_FOGO) .. "... um poder ancestral.", npc, player)
        elseif elemento == COMBAT_ICEDAMAGE then
            npcHandler:say("O " .. c("gelo", COR.ELEM_GELO) .. " na sua arma... respeite o frio, Aventureiro.", npc, player)
        end
    end

    -- Comenta Arremesso de Escudo
    local temEscudo = hasShield and hasShield(player)
    local temArma = weapon and weapon:getType():getWeaponType() ~= WEAPON_SHIELD
    if temEscudo and not temArma then
        npcHandler:say("Você luta apenas com o " .. c("escudo", COR.DOM_ESCUDO) .. "? Conheci um anão que arremessava o dele como uma rocha.", npc, player)
    end

    return true
end
npcHandler:setCallback(CALLBACK_MESSAGE_DEFAULT, creatureSayCallback)

npcHandler:setMessage(MESSAGE_FAREWELL, "Volte quando precisar de um bom metal.")
npcHandler:addModule(FocusModule:new(), npcConfig.name, true, true, true)
npcType:register(npcConfig)

🎨 6. USO DO SISTEMA DE CORES EM DIÁLOGOS (v9.1)

A tabela COR (v9.1) e a função c() são globais (definidas em texto_colorido.lua). Regras para NPCs:

Destaque de opções: c("{Aceitar}", COR.NPC_INTENCAO)

Comandos: c("{Senha}", COR.NPC_COMANDO)

Itens: c("{Lágrimas de Sereia}", COR.NPC_ITEM)

Locais: c("{Lago dos Reflexos}", COR.NPC_LOCAL)

Emoções: c("preocupado", COR.NPC_EMOCAO)

Missões: c("{missão}", COR.MISSAO_MATAR) ou conforme o tipo.

Domínios: use as constantes COR.DOM_* (ex.: COR.DOM_COMBATE_LAMINAS, COR.DOM_COMBATE_DESARMADO) e também as cores elementais (COR.ELEM_FOGO, etc.) para coerência com o HUD.

Exemplo de fala completa:

lua
npcHandler:say("Você encontrou as " .. c("{5 Lágrimas de Sereia}", COR.NPC_ITEM) .. " no " .. c("{Lago dos Reflexos}", COR.NPC_LOCAL) .. "?", npc, player)
Importante: enviarMsgColorida(player, mensagem) envia para o canal 24 (mensagens de sistema do SPA). Para o diálogo do NPC, prefira npcHandler:say com texto contendo {chaves} e, quando apropriado, cores inseridas com c() dentro da string.

👤 8. PERSONALIZAÇÃO POR PRONOME (VIA PRONOUNS.LUA)

A função NpcUtils.getTratamento deve usar Pronouns.getSubject(player) para determinar o tratamento correto, em vez de depender de storage 100001.

Exemplo de implementação interna de getTratamento:

lua
function NpcUtils.getTratamento(player)
    local subject = Pronouns.getSubject(player)
    if subject == "she" then
        return { artigo = "a", vocativo = "senhora", sufixoVoc = "a" }
    elseif subject == "he" then
        return { artigo = "o", vocativo = "senhor", sufixoVoc = "o" }
    else
        return { artigo = "", vocativo = "viajante", sufixoVoc = "e" }
    end
end
Uso nos diálogos:

lua
local trat = NpcUtils.getTratamento(player)
npcHandler:say("Ah, " .. trat.artigo .. " nobre " .. trat.vocativo .. " voltou! Já sabia que podia confiar em si.", npc, player)
Este sistema garante que o NPC se refira ao personagem com o pronome correcto, flexionando artigos e vocativos de forma natural. O assistente deve usar getTratamento sempre que o texto exigir concordância de género que dependa do pronome do personagem.

🧬 10. PERSONALIDADE DO NPC E PROGRESSÃO DE INTIMIDADE

Cada NPC no MCR tem uma personalidade definida. Níveis de intimidade (usando storages 100010+):

0: Desconhecido

1: Conhecido

2: Amigo

3: Confidente

Exemplos de personalidades:

Sábio caloroso: pergunta o pronome logo no primeiro encontro.

Guerreiro rude: usa termos neutros, não pergunta.

Místico enigmático: pergunta após intimidade 2.

Comerciante astuto: pergunta subtilmente no meio do diálogo.

A progressão de intimidade deve reflectir‑se na linguagem:

Intimidade 0: "Viajante, preciso de ajuda."

Intimidade 1: "Ah, o nobre senhor voltou! Já sabia que podia confiar em si."

Intimidade 3: "Meu caríssimo amigo! Sempre há lugar para si na minha mesa."

🔑 11. DESTAQUE DE PALAVRAS COM {CHAVES} E CORES

O OTClient Redemption reconhece palavras entre {chaves} como destaque visual (geralmente azul ou sublinhado), herança do protocolo Tibia. Agora, com o sistema de cores do SPA v4.2, podemos envolver a chave com c() para dar ainda mais significado.

Convenção MCR:

Ações: c("{aceitar}", COR.NPC_INTENCAO), c("{entregar}", COR.NPC_COMANDO), c("{missão}", COR.MISSAO_MATAR)

Itens: c("{5 Lágrimas de Sereia}", COR.NPC_ITEM)

Locais: c("{Lago dos Reflexos}", COR.NPC_LOCAL)

Recompensas: c("{500 pontos de experiência}", COR.RARIDADE_OURO)

⚠️ NUNCA use "}?" junto; sempre "} ?" para evitar erro visual no cliente.

Exemplo correcto: "Você " .. c("{aceita}", COR.NPC_INTENCAO) .. " ?"
Exemplo INCORRECTO: "Você {aceita}?"

🌠 12. REACÇÃO NARRATIVA AOS DOMÍNIOS DO AVENTUREIRO (SPA v4.2)

Os NPCs do Projeto MCR estão conscientes dos Domínios que o Aventureiro despertou e do seu progresso. Esta percepção é puramente narrativa e NUNCA expõe números, níveis ou termos técnicos. O NPC reage de forma natural, como alguém que nota a evolução de um artesão, de um guerreiro ou de um estudioso.

12.1 COMO OBTER OS DADOS DO JOGADOR

Nos scripts Lua, utilize os métodos do SPA (todos implementados):

player:getDominioAfinidade(dominioId) — retorna a afinidade bruta (para uso interno).

getNivelPorAfinidade(afinidade) — função global do SPA; retorna o nível (0‑20) correspondente.

player:getDominioState(dominioId) — retorna 0 (normal), 1 (Lampejo) ou 2 (Vínculo).

hasShield(player) — retorna true se o jogador tem escudo equipado (para detectar Arremesso de Escudo).

As constantes dos Domínios estão definidas em constantes.lua (pasta SPA/core/) e são globais: DOMINIO_LAMINAS = 10, DOMINIO_ESPADAS_LEVES = 100, DOMINIO_ESCUDO = 15, DOMINIO_ARREMESSO_ESCUDO = 140, DOMINIO_FOGO_AR = 23, etc.

12.3 REGRAS PARA A REACÇÃO

NUNCA diga "Você atingiu o nível 15 em Fogo." Use metáforas e observações sensoriais.

A reacção deve ser condizente com a personalidade do NPC.

A progressão de intimidade pode desbloquear comentários mais profundos sobre os Domínios.

🔥 13. PERCEPÇÃO DOS LAMPEJOS, VÍNCULOS E MAESTRIA (ESTADOS DE ALMA)

Os Lampejos e Vínculos são manifestações visíveis da afinidade do Aventureiro. A Maestria (nível 20) torna o Vínculo permanente. Alguns NPCs, especialmente os de natureza mística ou muito observadores, podem notar quando o jogador está sob um Estado de Alma activo e comentá‑lo.

Exemplos narrativos:

Quando o jogador está em Vínculo com Fogo:
"Há faíscas nos seus olhos hoje, Aventureiro. O fogo está convosco."

Quando o jogador está em Lampejo com Lâminas Leves:
"Você move‑se com uma leveza diferente... Como se o ar abrisse caminho para os seus passos."

Quando o jogador atingiu a Maestria (Vínculo permanente):
"O " .. c("fogo", COR.ELEM_FOGO) .. " nunca mais o abandonará. Você é um verdadeiro " .. c("Mestre das Chamas", COR.RARIDADE_LENDARIO) .. "."

Quando o jogador perde um Vínculo (antes da Maestria):
"A sintonia com as lâminas se desfez... talvez o foco tenha mudado."

Regras:

Apenas NPCs com personalidade adequada (místicos, sábios, elementais) devem comentar Lampejos/Vínculos.

O comentário deve ser subtil e ocasional; NÃO deve ocorrer a cada interação (use um cooldown por sessão, ex.: storage temporário).

Nunca usar os termos "Lampejo" ou "Vínculo" no diálogo. Prefira descrições poéticas.

Para a Maestria, o NPC pode usar o título de "Mestre" e reconhecer a permanência do poder.

⚔️ 14. COMENTÁRIOS SOBRE POSTURAS DE COMBATE E ARREMESSO DE ESCUDO

NPCs instrutores de combate (como Thorne no tutorial) podem comentar a postura actual do Aventureiro, dando dicas narrativas. As Posturas dependem do fightMode nativo, que já é funcional. Os NPCs podem ler o fightMode do jogador via player:getFightMode() e comentar. Também podem detectar o uso exclusivo de escudo (Arremesso de Escudo) via hasShield(player) e ausência de arma.

Ao adoptar Ímpeto (1):
"Ah, vejo que prefere a agressividade! Cuidado para não se expor demais."

Ao adoptar Guarda (3):
"Postura defensiva... Sábio. Mas lembre‑se: uma parede parada acaba por ruir."

Ao ver o jogador apenas com escudo:
"Sem arma, apenas com o " .. c("escudo", COR.DOM_ESCUDO) .. "? Você luta como um verdadeiro defensor. Sabe arremessá‑lo?"

Ao alternar frequentemente:
"Você adapta‑se como a água. Isso é raro. A maioria fixa‑se num só estilo."

Estes comentários devem ser raros (ex.: uma vez a cada 10 mudanças de postura) para não poluir o chat, e apenas para NPCs especificamente voltados ao ensino do combate.
