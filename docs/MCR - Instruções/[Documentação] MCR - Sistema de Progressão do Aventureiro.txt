🎯 Objetivo deste Documento
Este documento descreve a arquitetura, as regras e a API do Sistema de Progressão do Aventureiro (SPA). Ele serve como referência para qualquer pessoa que for manter, expandir ou utilizar o sistema noutros contextos (NPCs, quests, loot, etc.).

O SPA substitui completamente as vocações tradicionais. Todo personagem é um Aventureiro (vocation = 0) e progride horizontalmente através de Domínios, acumulando Afinidade ao realizar ações no mundo.

⚙️ Funcionamento Geral
O jogador realiza ações (dano, kill, usar ferramentas, lançar magias, receber dano).

O sistema acumula Afinidade na especialidade correspondente, distribuindo 50% para a especialidade, 33% para o secundário e 17% para o primário.

O histórico de ações alimenta o cálculo de Lampejos e Vínculos (automáticos).

O Vínculo permanece ativo enquanto a regra se mantiver (>50% das últimas 30 ações). Após 15 segundos sem cumprir, expira.

As posturas de combate (Ímpeto, Equilíbrio, Guarda) alteram dinamicamente velocidade, dano e defesa com base na arma e escudo.

Conforme o jogador sobe de nível nas especialidades, desbloqueia habilidades automáticas. Estas disparam com certa probabilidade durante o combate, exibindo um emote e uma mensagem colorida no chat.

🎨 Sistema de Cores
Todas as mensagens do SPA usam a sintaxe nativa do OTClient Redemption: {Texto, #CorHex}.
A tabela de cores está em texto_colorido.lua e segue uma lógica semântica:

Categoria	Exemplo	Cor
Positivo/Buff	COR.SUCESSO_CURA	#3CB371
Dano físico	COR.DANO_FISICO	#FF7F50
Condição (sangramento)	COR.COND_SANGRAMENTO	#8B0000
Inimigo	COR.INIMIGO_MONSTRO	#FF6347
Aliado	COR.ALIADO_JOGADOR	#00FF00
Informação de sistema	COR.SISTEMA_INFO	#87CEEB
Funções disponíveis:

c(texto, cor) → retorna a string formatada {texto, cor}.

corDominio(domId) → devolve a cor associada a um domínio.

sendColoredString(player, textoComTags) → envia texto com cores para o jogador.

📡 API Exposta ao Lua – Métodos C++ Adicionados para o SPA
Todos os métodos abaixo foram adicionados ao código‑fonte do Canary e estão disponíveis nos scripts Lua através do objeto player (ou creature, quando indicado).

🧬 Domínios e Afinidade
Método	Descrição
player:getDominioAfinidade(dominioId)	Retorna a afinidade acumulada no domínio.
player:addDominioAfinidade(dominioId, pontos)	Adiciona pontos de afinidade ao domínio.
player:setDominioDespertado(dominioId, bool)	Marca o domínio como despertado (já foi usado pelo menos uma vez).
player:hasHabilidade(habilidadeId)	Verifica se o jogador conhece uma habilidade.
player:learnHabilidade(habilidadeId)	Aprende uma habilidade permanentemente (grava na BD).
🌊 Sintonia Natural (Estados de Alma)
Método	Descrição
player:getDominioState(dominioId)	Retorna o estado de alma do domínio: 0 (nenhum), 1 (Lampejo) ou 2 (Vínculo).
player:setDominioState(dominioId, state)	Define o estado de alma.
player:getAllActiveStates()	Retorna uma tabela com todos os domínios que têm Lampejo ou Vínculo ativo.
player:registerDominioAction(dominioId)	Regista uma ação no histórico para que os Lampejos/Vínculos possam ser calculados.
player:getActionHistory(maxCount)	Devolve as últimas ações registadas (cada uma com dominioId e timestamp).
player:clearActionHistory()	Limpa todo o histórico de ações.
⚡ Velocidade de Ataque Dinâmica
Método	Descrição
player:setAttackSpeedModifier(mod)	Define o multiplicador de velocidade de ataque (ex.: 0.85 = 15% mais rápido).
player:getAttackSpeedModifier()	Retorna o multiplicador atual.
💧 Regeneração de Vida e Mana Dinâmica
Método	Descrição
player:setManaRegenModifier(mod)	Multiplicador da regeneração de mana (1.0 = normal).
player:getManaRegenModifier()	Retorna o valor atual.
player:setHealthRegenModifier(mod)	Multiplicador da regeneração de vida.
player:getHealthRegenModifier()	Retorna o valor atual.
player:setManaRegenTicksModifier(mod)	Multiplicador do intervalo entre ticks de regeneração de mana.
player:getManaRegenTicksModifier()	Retorna o valor atual.
player:setHealthRegenTicksModifier(mod)	Multiplicador do intervalo entre ticks de regeneração de vida.
player:getHealthRegenTicksModifier()	Retorna o valor atual.
🎯 Alcance de Ataque Dinâmico
Método	Descrição
player:setRangeModifier(mod)	Define o alcance extra (em tiles) para ataques à distância.
player:getRangeModifier()	Retorna o alcance extra atual.
🕶️ Furtividade (Stealth)
Método	Descrição
player:setStealth(bool)	Ativa ou desativa o modo furtivo. Quando ativo, monstros só notam o jogador a 1 SQM.
player:isStealth()	Retorna true se o jogador está furtivo.
🛡️ Posturas de Combate – Empurrar Criaturas
Método	Descrição
player:setCanPushCreatures(bool)	Permite que o jogador empurre criaturas "pushable" ao andar.
player:canPushCreatures()	Retorna true se a capacidade está ativa.
🩸 Dano Recebido Dinâmico
Método	Descrição
player:setDamageTakenModifier(mod)	Multiplicador do dano recebido (ex.: 0.85 = 15% de redução).
player:getDamageTakenModifier()	Retorna o multiplicador atual.
💀 Leech (Roubo de Vida e Mana)
Método	Descrição
player:setLifeLeechModifier(percent)	Percentagem do dano causado que é convertida em cura.
player:getLifeLeechModifier()	Retorna o valor atual.
player:setManaLeechModifier(percent)	Percentagem do dano causado que é convertida em mana.
player:getManaLeechModifier()	Retorna o valor atual.
💥 Knockback (Empurrar Inimigos)
Método	Descrição
creature:knockback(direction)	Empurra a criatura na direção fornecida. Retorna true se o movimento foi bem‑sucedido.
(Nota: a direção é um número de 0 a 3, representando Norte, Leste, Sul, Oeste.)

🔍 Checklist de Revisão para Novas Funcionalidades
A mensagem usa o formato {Texto, #Cor}.

As cores são escolhidas da tabela COR com significado adequado.

O feedback não polui o chat (emotes + log branco/amarelo).

As palavras de ligação herdam a cor do canal (sem c()).

O código está em português e salvo como ISO‑8859‑1.
