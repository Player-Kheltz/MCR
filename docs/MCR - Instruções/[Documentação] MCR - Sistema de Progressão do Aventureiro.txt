📘 Guia do Sistema de Progressão do Aventureiro (SPA)
Projeto MCR — Versão 3.2.0
Autor: Kheltz (Criador do Projeto MCR)
Última actualização: 17 de Maio de 2026

🌌 Apresentação para o Jogador
Bem‑vindo, aventureiro.
Em MCR, não existem vocações fixas. Tu não és apenas um Knight, um Paladin ou um Sorcerer.
És um Aventureiro — e o teu caminho é moldado pelas tuas acções, não por uma classe escolhida no início.

O Sistema de Progressão do Aventureiro (SPA) substitui completamente as vocações tradicionais.
Cada arma que empunhas, cada magia que lanças, cada escudo que levantas — tudo contribui para o teu domínio pessoal sobre o mundo.

🧭 1. Como Funciona a Progressão
Sempre que realizas uma acção — atacar, matar criaturas, usar ferramentas, lançar magias, até mesmo receber dano — o SPA regista essa acção e concede‑te Afinidade num Domínio relacionado.

Atacar com uma espada → ganhas afinidade em Espadas Leves ou Pesadas.

Atacar com um machado → afinidade em Machados.

Usar uma varinha mágica → afinidade em Wands.

Receber dano → afinidade em Sobrevivência.

Conforme a tua afinidade cresce, tu ganhas níveis nesse domínio.
Cada nível desbloqueia novas habilidades e melhora as existentes, tornando‑te progressivamente mais poderoso naquele estilo de jogo.

🗺️ 2. Os Domínios
Os domínios estão organizados em três camadas:

Camada	Descrição	Exemplo
Primário	O mais abrangente — beneficia todos os estilos de combate ou magia.	Combate, Magia
Secundário	O tipo de arma ou magia.	Lâminas, Machados, Runologia
Especialidade	A variante exacta: leve, pesada, com ou sem escudo.	Espadas Leves, Arremesso, Desarmado Pesado
Afinidade ganha num domínio espalha‑se automaticamente para os seus pais:
50% para a especialidade → 33% para o secundário → 17% para o primário.

Isto significa que, ao treinares Espadas Leves, também melhoras Lâminas e Combate, mas o foco principal está sempre na arma que realmente usas.

⚔️ 3. Posturas de Combate
A qualquer momento podes alternar entre três posturas:

Postura	Filosofia	Efeito Geral
Ímpeto	Ataque total	Mais dano, mais velocidade, mas menos defesa. As habilidades de ataque tornam‑se mais agressivas.
Equilíbrio	Versatilidade	Valores intermédios. Podes usar tanto habilidades de ataque como de defesa, mas sem os extremos de Ímpeto ou Guarda.
Guarda	Defesa focada	Menos dano, mas mais defesa, regeneração e efeitos de controlo. As habilidades defensivas brilham aqui.
A postura altera o comportamento das tuas habilidades.
Uma habilidade de sangramento em Ímpeto pode causar mais dano; em Guarda, pode durar o dobro do tempo.

Além disso, a postura interage com o teu equipamento:

Armas de duas mãos → estilo Pesado (mais dano, knockback).

Armas de uma mão sem escudo → estilo Leve (mais velocidade).

Com escudo → benefícios defensivos adicionais.

✨ 4. Lampejos e Vínculos (Sintonia Natural)
Quanto mais usas um domínio, mais "sintonizado" ficas com ele.
O SPA monitoriza as tuas acções recentes e pode conceder‑te estados especiais:

Estado	Condição	Efeito
Lampejo	3 das últimas 5 acções são do mesmo domínio.	Bónus moderado de velocidade e dano. Dura 8 segundos.
Vínculo	Mais de 50% das últimas 30 acções são do mesmo domínio.	Bónus significativo de velocidade e dano. Mantém‑se enquanto continuares a usar o domínio.
Estes estados são silenciosos — não há mensagens no chat (excepto quando um Vínculo se forma ou quebra).
Tu sente‑los através da fluidez dos teus ataques e do poder dos teus golpes.

🛡️ 5. Habilidades — A Essência do Teu Estilo
Ao ganhares níveis nos domínios, desbloqueias habilidades automáticas.
Estas habilidades disparam durante o combate, com uma probabilidade que aumenta com os teus Lampejos e Vínculos.

Cada habilidade evolui contigo.
Uma habilidade aprendida no nível 5 de um domínio será fraca no início, mas tornar‑se‑á devastadora quando atingires a Maestria (nível 20).
Se aprenderes uma habilidade mais cedo (por exemplo, através de um NPC ou quest), ela funcionará na mesma — mas com o poder correspondente ao teu nível actual.

As habilidades respeitam o feedback visual do Tibia:

Cada golpe extra aparece como uma linha de dano no chat.

Projécteis (flechas, lanças, bolas de energia) são enviados visualmente.

Condições como sangramento, atordoamento e veneno usam os efeitos nativos do jogo.

Nada de "dano invisível" — tudo o que o SPA faz é real e visível.

🎨 6. Cores e Feedback
O SPA usa um sistema de cores para te manter informado sem poluir o chat:

Cor	Significado	Exemplo
🟢 Verde	Cura, sucesso	"Você aprendeu Hemorragia!"
🔵 Azul	Informação do sistema	"Sua afinidade com Lâminas se aprofunda."
🟠 Laranja	Avisos de combo	"[Disparo Duplo] Duas flechas partem em uníssono!"
🔴 Vermelho	Dano, inimigos	"Centopeia" (nome do monstro)
🟡 Dourado	Maestria, conquistas	"Você se tornou um Mestre das Lâminas!"
As palavras‑chave (nomes de domínios, monstros, habilidades) são sempre coloridas.
O texto narrativo permanece na cor padrão do canal, para não cansar os olhos.

🧬 2. Tabelas da Base de Dados
2.1 dominios_def
Armazena a definição de todos os domínios.

Coluna	Tipo	Descrição
id	smallint UNSIGNED	Identificador único do domínio (ex.: 100, 15).
nome	varchar(50)	Nome legível.
descricao	text	Descrição opcional.
2.2 player_dominios
Regista a afinidade de cada jogador por domínio.

Coluna	Tipo	Descrição
player_id	int(11)	FK para players.id.
dominio_id	smallint UNSIGNED	FK para dominios_def.id.
afinidade	int(11)	Pontos acumulados.
despertado	tinyint(1)	Se o domínio já foi despertado (1 = sim).
Chave primária composta: (player_id, dominio_id).

2.3 habilidades_def
Definição estática das habilidades (usada para documentação e pelo cliente, se necessário).

Coluna	Tipo	Descrição
id	int	ID único da habilidade.
nome	varchar(50)	Nome da habilidade.
dominio_id	smallint	Domínio principal associado.
tipo	enum('passiva','gatilho','transformacao')	Tipo de habilidade.
requisito_afinidade	int	Nível mínimo do domínio para aprender.
descricao	text	Descrição curta do efeito.
2.4 player_habilidades
Habilidades que o jogador já aprendeu.

Coluna	Tipo	Descrição
player_id	int(11)	FK para players.id.
habilidade_id	int	FK para habilidades_def.id.
Chave primária composta: (player_id, habilidade_id).

⚙️ 5. Fluxo de Dados Detalhado
5.1 Ataque Físico
ProgDano (eventos.lua) captura onDealtDamage.

Identifica o domínio da arma via getWeaponEspecialidade.

Chama adicionarAfinidade(player, dom, pontos).

Acumula pontos em STORAGE.AFINIDADE_BUFFER + dom.

Chama player:registerDominioAction(dom).

Chama onDominioAction(player, dom) (sintonia.lua) para avaliar Lampejos/Vínculos.

Chama executarHabilidades(player, "onAttack", target, damage, primary) (executor_habilidades.lua).

Verifica habilidades de gatilho compatíveis, condições, chances e aplica efeitos.

PosturaDmg (postura.lua) processa knockback, furtividade, applyPendingCondition e processBuffs.

5.2 Ciclo Periódico (SintoniaThink, a cada 2s)
descarregarBuffer(player) (progressao.lua):

Lê todos os buffers 95001 + dom.

Chama player:addDominioAfinidadeBatch(dados).

Para cada domínio com ganhos, chama verificarDesbloqueio.

Expira Lampejos cujo tempo acabou.

Verifica Vínculos que não tiveram ação recente.

Chama aplicarBonus(player):

Calcula bónus de velocidade/dano a partir dos estados activos.

Se os valores mudaram (comparação com storages actuais), actualiza 90004/90005 e chama aplicarPostura.

5.3 Logout
ProgressaoLogout (login.lua) chama descarregarBuffer(player).

PosturaLogout (postura.lua) limpa modificadores e storages.

🎨 6. Sistema de Cores
A tabela COR (em texto_colorido.lua) contém constantes de cores no formato #RRGGBB. A função c(texto, cor) retorna {texto, cor}, que é interpretado pelo OTClient Redemption.
A função enviarMsgColorida(player, texto) envia a string formatada para o canal 24 (MESSAGE_INFO_DESCR).
O canal 23 (MESSAGE_STATUS_WARNING) é usado para feedback de combate.
