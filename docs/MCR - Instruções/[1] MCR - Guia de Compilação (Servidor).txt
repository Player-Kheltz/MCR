🎯 OBJETIVO DESTE GUIA
Fornecer todas as instruções, dependências, customizações e verificações necessárias para compilar o servidor Canary do Projeto MCR a partir do código‑fonte. Esta versão incorpora todos os métodos Lua exigidos pelo Sistema de Progressão do Aventureiro (SPA v4.5.0), incluindo gerenciamento de afinidades em lote, histórico de ações (para Lampejos, Vínculos e Maestria), knockback, modificadores de velocidade e stealth, getAmmo (para projéteis dinâmicos), getVocationBaseMaxHealth, getVocationBaseMaxMana, getNaturalSpeed, setMovementSpeed e setCombatModifier (modificadores de combate dinâmicos), além da estrutura modular de scripts em data-canary/scripts/MCR/SPA/core/ e subpastas. O assistente DEVE seguir estas directrizes ao modificar o código C++.

🚧 ESTADO ATUAL DA IMPLEMENTAÇÃO

Todos os métodos C++ descritos neste guia já estão implementados e funcionais no código‑fonte do servidor. Não há mais dependências em fases futuras. Os scripts da pasta MCR/SPA/ utilizam‑nos plenamente.

⚠️ REGRAS DE REVISÃO PARA O ASSISTENTE (C++ – SERVIDOR CANARY)

Ao modificar QUALQUER arquivo .cpp/.h no diretório Canary/src/, o assistente DEVE:

Garantir que TODAS as strings visíveis ao jogador (sendTextMessage, sendCancelMessage, logs de sistema) estejam em PT‑BR, com acentuação correcta. Os caracteres acentuados DEVEM ser escritos OBRIGATORIAMENTE como escapes octais Latin‑1 (ex.: "N\355vel", "Voca\347\343o"). NÃO é permitido usar UTF‑8 directamente nas strings; usar escapes octais garante portabilidade e evita problemas de compilador.

Manter comentários em PT‑BR, excepto em nomes de funções/variáveis que permanecem em inglês (camelCase ou snake_case).

Verificar se a alteração mantém compatibilidade com o protocolo 15.00 e não quebra bindings Lua existentes (registerMethod, registerClass).

Se adicionar novos métodos expostos ao Lua, seguir a convenção de nomenclatura minúsculas_com_underscore e incluir validação de parâmetros (número e tipo). Documentar cada novo método neste guia.

Se alterar comunicação de rede (ProtocolGame), garantir que opcodes estendidos estejam na faixa reservada (160–255) e sejam documentados.

Toda nova consulta SQL DEVE usar placeholders parametrizados (evitar concatenação).

Após a edição, listar explicitamente os arquivos modificados para compilação selectiva.

O sistema de progressão usa Revscript: os scripts Lua ficam na pasta data-canary/scripts/MCR/SPA/. O motor carrega‑os automaticamente; não é necessário alterar o C++ para registar scripts.

📦 DEPENDÊNCIAS (AMBIENTE WINDOWS)

Ferramentas Obrigatórias:

Visual Studio 2022 (Community ou superior) com carga de trabalho "Desenvolvimento para desktop com C++".

CMake 3.20 ou superior (adicionar ao PATH).

Git for Windows.

vcpkg (gerenciador de bibliotecas). Instale em C:\vcpkg (ou caminho sem espaços).

Comandos de instalação do vcpkg:
git clone https://github.com/Microsoft/vcpkg.git C:\vcpkg
cd C:\vcpkg
.\bootstrap-vcpkg.bat
.\vcpkg integrate install

Bibliotecas Requeridas (instalar via vcpkg, para x64-windows):
.\vcpkg install boost-beast:x64-windows boost-asio:x64-windows
.\vcpkg install boost-filesystem:x64-windows boost-system:x64-windows
.\vcpkg install boost-variant:x64-windows libmariadb:x64-windows
.\vcpkg install lua:x64-windows pugixml:x64-windows cryptopp:x64-windows
.\vcpkg install fmt:x64-windows spdlog:x64-windows nlohmann-json:x64-windows
.\vcpkg install gtest:x64-windows (opcional, para testes)

Observação: os pacotes podem mudar de nome; consulte o README.md do repositório canary para obter a lista exacta para a versão actual.

🔨 PROCEDIMENTO DE COMPILAÇÃO (PADRÃO)

Clone o repositório MCR (já contém os fontes customizados):
git clone https://github.com/Player-Kheltz/ProjetoMCR.git
cd ProjetoMCR\Canary

Configure o CMake (substitua o caminho do vcpkg pelo seu):
mkdir build
cd build
cmake .. -DCMAKE_TOOLCHAIN_FILE=C:/vcpkg/scripts/buildsystems/vcpkg.cmake -DVCPKG_TARGET_TRIPLET=x64-windows

Compile:
cmake --build . --config Release

O executável será gerado em build\Release\canary.exe.

Copie o executável para a raiz do servidor (onde estão data/, config.lua) ou configure o output directamente com cmake --install . (se configurado).

🚀 Novos Métodos Lua para o SPA v4.5.0 (IMPLEMENTADOS)

Estes métodos estão implementados em player.h/cpp e expostos em luascript.cpp. São essenciais para o funcionamento do buffer de progressão, Lampejos, Vínculos, Maestria, Posturas, Sinergia Elemental, Arremesso de Escudo, habilidades, e para os novos sistemas de vida, mana e velocidade sem acumulação. Todos estão activos e podem ser usados pelos scripts da pasta MCR/SPA/.

player:getDominioAfinidade(dominioId) – retorna a afinidade acumulada no Domínio.

player:addDominioAfinidade(dominioId, valor) – adiciona afinidade (INSERT ... ON DUPLICATE KEY UPDATE).

player:addDominioAfinidadeBatch(tabela) – adiciona afinidades em lote, recebendo uma tabela {[dominioId] = pontos}. Optimizado para múltiplas actualizações.

player:getDominioAfinidades(dominio1, dominio2, ...) – consulta várias afinidades de uma vez, retornando uma tabela {[dominioId] = valor}.

player:setDominioDespertado(dominioId, bool) – marca o Domínio como despertado (1/0).

player:hasHabilidade(habilidadeId) – verifica se o jogador conhece a habilidade (consulta a BD).

player:learnHabilidade(habilidadeId) – concede uma habilidade permanentemente (insere na BD).

player:getHabilidades() – retorna uma tabela com os IDs de todas as habilidades aprendidas.

player:registerDominioAction(dominioId) – regista uma acção no histórico do jogador (usado para Lampejos/Vínculos/Maestria).

player:getActionHistory(maxCount) – retorna as últimas acções como tabela de {dominioId, timestamp}.

player:clearActionHistory() – limpa todo o histórico de acções.

player:getDominioState(dominioId) – retorna o estado de alma: 0 = normal, 1 = Lampejo, 2 = Vínculo (Maestria garante Vínculo permanente).

player:setDominioState(dominioId, state) – define o estado (0,1,2).

player:getAllActiveStates() – retorna uma tabela {[dominioId] = state} para domínios com estado > 0.

player:setAttackSpeedModifier(mod) – multiplicador da velocidade de ataque (0.1 a infinito).

player:getAttackSpeedModifier() – retorna o valor actual.

player:setStealth(bool) – activa/desactiva furtividade (invisibilidade condicional).

player:isStealth() – verifica se está furtivo.

player:setCanPushCreatures(bool) – permite empurrar criaturas ao andar.

player:setLifeLeechModifier(percent) – define % de life leech.

player:setManaLeechModifier(percent) – define % de mana leech.

player:setDamageTakenModifier(mod) – multiplicador do dano recebido.

player:setRangeModifier(tiles) – ajusta o alcance do jogador.

creature:knockback(direction) – empurra a criatura na direcção (0‑3, mesma orientação do jogo).

player:setStorages({[key] = value, ...}) – escreve múltiplas storages de uma vez.

player:getStorages(key1, key2, ...) – lê múltiplas storages, retorna tabela {[key] = value}.

player:getAmmo() – retorna o item da munição real que o jogador está a usar (procura no slot de munição e dentro de aljavas/quivers). Essencial para o sistema de projéteis dinâmicos.

player:getParty() – retorna o grupo do jogador (ou nil).

monster:getRaceId() – retorna o ID da raça do monstro.

player:setCombatModifier("chave", valor) – define um modificador de combate dinâmico (ex.: "fire_damage", 25.0).

player:getCombatModifier("chave") – retorna o valor actual do modificador.

item:getElementalBond() – retorna o tipo de dano elemental do item (COMBAT_FIREDAMAGE, etc.).

item:getElementType() – retorna o elementType do item (wandType, combatType do Abilities).

player:unlearnHabilidade(habilidadeId) – Remove uma habilidade específica da BD (DELETE individual).

player:unlearnAllHabilidades() – Remove todas as habilidades do jogador da BD (uma única query).

player:learnHabilidadeBatch(tabela) – Aprende várias habilidades em lote (INSERT em batch).

player:getVocationBaseMaxHealth() – Retorna a vida base calculada a partir da vocação Aventureiro (gainhp * level + offset). Essencial para o cálculo de passivas de vida sem acumulação.

player:getVocationBaseMaxMana() – Retorna a mana base calculada a partir da vocação Aventureiro (gainmana * level + offset). Essencial para o cálculo de passivas de mana sem acumulação.

player:getNaturalSpeed() – Retorna a velocidade natural do personagem (base da vocação + bónus por nível). Usada como referência para aplicar multiplicadores de velocidade de movimento.

player:setMovementSpeed(speed) – Define a velocidade de movimento atual do personagem, atualizando o cliente imediatamente.

⚙️ Lógica interna já presente em player.cpp:

std::deque<ActionRecord> para o histórico de acções.

Estados de alma (Lampejo/Vínculo) controlados via setDominioState e getAllActiveStates, consumidos pelos scripts de sintonia.

Maestria (nível 20) gerida via getNivelPorAfinidade e setDominioState: quando o nível atinge 20, o Vínculo torna‑se permanente e não é removido pela sintonia.

Buffer de afinidade gerido via storages temporárias (95001+dominioId); o descarregamento em lote usa addDominioAfinidadeBatch para minimizar queries.

Modificadores de combate dinâmicos armazenados em mapa string→double, manipulados via setCombatModifier/getCombatModifier.

Vida e mana base calculadas com getVocationBaseMaxHealth e getVocationBaseMaxMana, utilizando offsets para garantir os valores iniciais corretos (150 de vida e 50 de mana no nível 1).

Velocidade natural calculada via getNaturalSpeed (base + level - 1), utilizada como referência limpa para multiplicadores das passivas de movimento.

🛠️ PIPELINE DE TRADUÇÃO AUTOMÁTICA

Strings de interface e logs: PT‑BR com escapes octais Latin‑1 (OBRIGATÓRIO). NUNCA colocar texto acentuado em UTF‑8 directamente nas strings C++. Exemplo correcto: "N\355vel de magia \351 alto." Exemplo INCORRECTO: "Nível de magia é alto."

Bindings Lua intactos: nomes de funções registadas NUNCA são alterados.

Cabeçalhos (.h/.hpp): comentários traduzidos, definições de classes/macros intactas.

⚠️ ARMADILHAS E LIÇÕES APRENDIDAS

Codificação de console: SetConsoleOutputCP(CP_UTF8) é imprescindível para logs em PT‑BR.

Tradução de logs: use SEMPRE escapes octais para todos os acentos nas strings C++. Não confie em UTF‑8 directo.

Conexão com MariaDB: verificar senha em config.lua e se o serviço está rodando.

Compilação com vcpkg: mantenha o vcpkg atualizado.

Escapes octais vs UTF‑8: use sempre escapes octais para portabilidade. Esta é uma regra absoluta no MCR.

A coluna lastlogin EXISTE na tabela players do schema actual do Canary (verificado no banco de dados do projecto). Pode ser usada para lógica de inicialização/migração de personagens no evento_login.lua.

O SPA v4.5 exige que o buffer de afinidade seja descarregado periodicamente; o método addDominioAfinidadeBatch deve ser eficiente para evitar lag.

A função player:getAllActiveStates() deve ser usada para obter os estados de alma; não itere manualmente sobre todos os domínios.

As cores definidas em texto_colorido.lua (v10.0) devem ser mantidas consistentes com as constantes usadas nos scripts do servidor.

O método player:getAmmo() é essencial para o sistema de projéteis dinâmicos e deve ser preservado.

Os métodos player:setCombatModifier e getCombatModifier suportam a Sinergia Elemental e outras mecânicas dinâmicas do SPA v4.5.

Os novos métodos getVocationBaseMaxHealth, getVocationBaseMaxMana, getNaturalSpeed e setMovementSpeed são utilizados pelas funções centrais de atributos (recalcularVidaMana e recalcularVelocidade) e NÃO devem ser alterados, pois garantem a estabilidade dos valores sem acumulação.
