PROJETO MCR – HABILIDADE GABARITO (Documento Canónico)
Versão 4.5 – Sistema de Progressão do Aventureiro (SPA v4.3)
Arquivo: [Gabarito] Habilidade Gabarito.txt

🎯 OBJECTIVO DESTE DOCUMENTO
Este documento é a referência máxima para a criação de qualquer habilidade
no Projeto MCR. Define todos os campos possíveis, as regras de feedback
narrativo, os pacotes temáticos, as directrizes de implementação e as
convenções que garantem a coerência e a qualidade do SPA.

A partir da versão 4.2, o SPA adopta uma arquitectura modular em três camadas:
  - Motor de Habilidades (motor_habilidades.lua) – lógica de execução.
  - Definições de Habilidades (dados/*.lua) – apenas parâmetros, zero lógica.
  - Narrativa (narrativa_habilidades.lua) – textos de feedback.

Sempre que um criador de conteúdo ou assistente for gerar uma nova
habilidade, DEVE seguir este gabarito.

================================================================================
📦 ESTRUTURA GERAL DE UMA HABILIDADE
================================================================================

Cada habilidade é uma entrada na tabela global HABILIDADES[habId] = { ... }.
Os campos estão organizados em:

  [O]  = Obrigatório
  [C]  = Condicional (obrigatório apenas para certos tipos)
  [OP] = Opcional

================================================================================
🧩 ARQUITECTURA MODULAR (NOVO)
================================================================================

A partir da v4.2, as habilidades de gatilho não contêm lógica de execução
no campo 'efeito'. Em vez disso, usam o campo 'efeitoConfig', que é uma
tabela de parâmetros interpretada pelo motor central (motor_habilidades.lua).

Vantagens:
  - Novas habilidades são criadas apenas com parâmetros, sem código repetitivo.
  - O feedback narrativo é gerado automaticamente pela Narrativa, sem
    necessidade de escrever mensagens manuais (a menos que se queira).
  - Alterações no motor propagam-se para todas as habilidades existentes.

================================================================================
🏷️ CAMPOS OBRIGATÓRIOS
================================================================================

nome
  [O] Nome da habilidade. Aparece como emote sobre o jogador
      (TALKTYPE_MONSTER_SAY) e nos menus de diagnóstico.
  Exemplo: "Investida Brutal"

tipo
  [O] Tipo da habilidade:
      "passiva"        – efeito permanente, aplicado no login e renovado
                         a cada 2 segundos pelo think_master.
      "gatilho"        – ativada por eventos: onAttack, onDealtDamage,
                         onKill, onBlock, onReceiveDamage.
      "transformacao"  – activa/desactiva com base nos Estados de Alma
                         (Lampejo/Vínculo/Maestria). Suporta o campo
                         'desfazer' para reverter os bónus.

dominio
  [O] Lista de IDs de domínios onde esta habilidade se aplica.
      Exemplo: {111} = só Machados Pesados; {110, 111} = Machados Leves
      e Pesados.

nivelMin
  [O] Nível mínimo do domínio principal para aprender a habilidade
      automaticamente. Se aprendida antes (por quest/NPC), funciona
      mas com valores base (nível 1).

descricaoEfeito
  [O] Descrição curta do efeito. Aparece APENAS ao aprender a habilidade
      (chat colorido). Pode ser string fixa ou função(player) que retorna
      uma string.

cor
  [O] Cor base da habilidade, usada no emote, nos colchetes da mensagem
      e na mensagem de aprendizagem. Usar SEMPRE a cor do domínio
      principal (COR.DOM_COMBATE_* ou COR.DOM_MAGIA_*).

mensagem
  [O] Mensagens de feedback narrativo (canal 24).
      Com a nova arquitectura modular, este campo é OPCIONAL para
      habilidades de gatilho que usam 'efeitoConfig' – a Narrativa
      gera o feedback automaticamente.
      Se quiseres mensagens personalizadas, aplicam-se as mesmas regras:
      Para cada postura [1]=Ímpeto, [2]=Equilíbrio, [3]=Guarda:
        Pode ser uma tabela de funções (player, target) que retornam a
        string formatada.
        Pode ser uma única função (player, target).
        Pode ser uma tabela de strings estáticas (para passivas e
        transformações).
      Se não houver entrada para uma postura, o executor usa [2].

efeitoConfig
  [O para gatilhos modulares] Tabela de parâmetros que o motor central
      interpreta para executar o efeito.
      Deve conter obrigatoriamente o campo 'tipo', que define qual efeito
      será aplicado. Os restantes campos dependem do tipo escolhido.
      Ver secção "TIPOS DE EFEITOS DO MOTOR" abaixo.

================================================================================
💬 REGRAS DE FEEDBACK LIMPO (v3.3)
================================================================================

FORMATOS POR TIPO DE HABILIDADE:

  • Ataque SEM efeito adicional:
    "[Nome] Frase narrativa normal com o Alvo colorido."
    Exemplo: c("[Estocada] ", COR) .. "Uma estocada rápida atinge "
             .. c(t:getName(), COR.INIMIGO_MONSTRO) .. "!"

  • Ataque COM efeito (sangramento, atordoamento, knockback, veneno, etc.):
    "[Nome] Frase curta em Alvo - Efeito."
    Exemplo: c("[Estocada] ", COR) .. "Estocada precisa em "
             .. c(t:getName(), COR.INIMIGO_MONSTRO) .. " - "
             .. c("Sangramento", COR.COND_SANGRAMENTO) .. "."

  • Defesa (onBlock / onReceiveDamage):
    "[Nome] Frase narrativa sobre o bloqueio/defesa contra Alvo - Efeito."
    Exemplo: c("[Contra-Ataque] ", COR) .. "O escudo bloqueia o golpe de "
             .. c(t:getName(), COR.INIMIGO_MONSTRO) .. " - "
             .. c("Contra-atacado", COR.ACAO_KNOCKBACK) .. "."

  • Múltiplos efeitos:
    "[Nome] Frase narrativa em Alvo - Efeito1, Efeito2 e Efeito3."
    Exemplo: c("[Dilacerar] ", COR) .. "O machado dilacera "
             .. c(t:getName(), COR.INIMIGO_MONSTRO) .. " - "
             .. c("Sangramento", COR.COND_SANGRAMENTO) .. " e "
             .. c("Armadura quebrada", COR.ACAO_KNOCKBACK) .. "."

REGRAS ABSOLUTAS:
  - Começar SEMPRE com c("[Nome da Habilidade] ", corDoDominio).
  - Usar c(t:getName(), COR.INIMIGO_MONSTRO) para o nome do alvo.
  - Apenas o NOME DO ALVO e 1‑2 EFEITOS (condições, acções) são coloridos.
  - O texto narrativo é NEUTRO (cor normal do chat).
  - NUNCA usar travessões longos (—) ou caracteres não‑ASCII de 1 byte.
    Usar hífen (-) e aspas rectas.
  - NUNCA colorir palavras descritivas como "lâmina", "sangue", "impacto".
    Apenas termos de efeitos mecânicos: "Sangramento", "Atordoado", etc.
  - Cores de efeitos: usar COR.COND_* para condições, COR.ACAO_* para acções
    físicas, COR.SUCESSO_CURA para curas, COR.DANO_CRITICO para dano massivo.

================================================================================
⚙️ TIPOS DE EFEITOS DO MOTOR (efeitoConfig.tipo)
================================================================================

Cada habilidade de gatilho que usa o motor modular deve definir um
campo 'efeitoConfig' com um 'tipo' da lista abaixo. Os restantes campos
dependem do tipo.

dano_extra
  - Dano adicional simples (corpo a corpo ou projétil).
  - Campos: percentual (0.3), tipo ("distancia"|nil), magicEffect
  - Exemplo: { tipo = "dano_extra", percentual = 0.25, magicEffect = CONST_ME_DRAWBLOOD }

area_ground
  - Explosão no chão, centrada no jogador ou no alvo.
  - Campos: percentual, centro ("jogador"|"alvo"), raio, magicEffect, knockback (bool), condicao (CONDITION_*), condDuration, periodicDamage
  - Exemplo: { tipo = "area_ground", centro = "jogador", raio = 1, percentual = 0.4 }

area_target
  - Explosão centrada no alvo, com dano extra ao alvo principal.
  - Campos: percentual, areaPercentual, raio, magicEffect, knockback (bool)
  - Exemplo: { tipo = "area_target", percentual = 0.5, areaPercentual = 0.3, raio = 2 }

ricochete
  - Projétil que salta do alvo principal para N inimigos próximos.
  - Campos: percentual, raio, saltos, magicEffect
  - Exemplo: { tipo = "ricochete", percentual = 0.4, raio = 2, saltos = 2 }

knockback
  - Empurra o alvo. Pode ou não causar dano adicional.
  - Campos: comDano (bool), percentual, magicEffect
  - Exemplo: { tipo = "knockback", comDano = true, percentual = 0.3 }

condicao
  - Aplica uma condição ao alvo, com ou sem dano adicional.
  - Campos: conditionType, duration, periodicDamage, comDano (bool), percentual, magicEffect
  - Exemplo: { tipo = "condicao", conditionType = CONDITION_BLEEDING, duration = 4000, periodicDamage = 15 }

buff_speed
  - Aceleração de ataque.
  - Campos: multiplier, durationMs
  - Exemplo: { tipo = "buff_speed", multiplier = 0.4, durationMs = 700 }

buff_damage
  - Aumento de dano por N ataques.
  - Campos: percentExtra, buffHits
  - Exemplo: { tipo = "buff_damage", percentExtra = 20, buffHits = 3 }

life_leech
  - Roubo de vida.
  - Campos: leechPercent, durationSec
  - Exemplo: { tipo = "life_leech", leechPercent = 15, durationSec = 4 }

mana_leech
  - Roubo de mana.
  - Campos: leechPercent, durationSec
  - Exemplo: { tipo = "mana_leech", leechPercent = 10, durationSec = 4 }

next_condition
  - Prepara uma condição para o próximo ataque.
  - Campos: conditionType, duration, periodicDamage
  - Exemplo: { tipo = "next_condition", conditionType = CONDITION_BLEEDING, duration = 3500, periodicDamage = 12 }

finisher
  - Dano massivo contra alvos com vida baixa (<20%).
  - Campos: percentual, magicEffect, knockback (bool), medoArea (bool), sangramentoArea (bool)
  - Exemplo: { tipo = "finisher", percentual = 1.5, medoArea = true }

defesa_cura
  - Recupera vida ao bloquear.
  - Campos: percentual, reflect (bool), reflectPercentual
  - Exemplo: { tipo = "defesa_cura", percentual = 0.04, reflect = true, reflectPercentual = 0.2 }

defesa_barreira
  - Reduz dano recebido temporariamente.
  - Campos: factor, durationMs
  - Exemplo: { tipo = "defesa_barreira", factor = 0.5, durationMs = 2000 }

defesa_contra_ataque
  - Contra-ataca todos os inimigos ao redor ao bloquear.
  - Campos: percentual, magicEffect
  - Exemplo: { tipo = "defesa_contra_ataque", percentual = 0.3 }

field
  - Cria um campo elemental no chão.
  - Campos: centro ("jogador"|"alvo"), elemento, danoPorTick, duracaoMs, intervaloMs, magicEffect
  - Exemplo: { tipo = "field", centro = "alvo", elemento = COMBAT_FIREDAMAGE, danoPorTick = 10, duracaoMs = 5000 }

sinergia
  - Combina múltiplos domínios. Usa os campos 'sinergiaDominios' e
    'efeitosSecundarios' da habilidade.
  - Campos: efeitoBase (tipo de efeito a aplicar primeiro), nivelMinSecundario
  - Exemplo: { tipo = "sinergia", efeitoBase = "area_target", nivelMinSecundario = 10 }

rajada
  - Dispara N projéteis em sequência contra o alvo.
  - Campos: percentual, numProjeteis, intervaloMs
  - Exemplo: { tipo = "rajada", percentual = 0.3, numProjeteis = 3, intervaloMs = 250 }

corrente
  - Liga dois alvos, causando dano ao secundário.
  - Campos: percentual, raio, magicEffect
  - Exemplo: { tipo = "corrente", percentual = 0.4, raio = 2 }

================================================================================
🎯 CAMPOS CONDICIONAIS (gatilho)
================================================================================

gatilho
  [C] Gatilho que activa a habilidade:
      "onAttack"         – ao causar dano.
      "onDealtDamage"    – após causar dano.
      "onKill"           – ao matar uma criatura.
      "onBlock"          – ao bloquear um ataque (exige escudo).
      "onReceiveDamage"  – ao receber dano (defesa genérica, não exige arma).

cooldown
  [C] Cooldown individual em segundos.

prioridade
  [C] Prioridade base (1‑10). Habilidades com maior prioridade são
      escolhidas primeiro pelo motor inteligente.

categoria
  [C] Categoria da habilidade:
      "single"       – dano directo elevado.
      "aoe"          – atinge múltiplos alvos.
      "debuff"       – aplica condição negativa.
      "buff"         – melhora o próprio jogador.
      "defense"      – cura, escudo, redução de dano.
      "chi_spender"  – consome Chi (requer chiCusto definido).
      "finisher"     – efeito extra quando alvo está com vida baixa.
      "sinergia"     – habilidade que escala com múltiplos domínios.
      "condicao"     – especializada em aplicar uma condição prolongada.

efeito
  [C] Função que executa o efeito principal.
      ⚠️ OBSOLETO para novas habilidades de gatilho. Usar 'efeitoConfig'.
      Mantido para compatibilidade com habilidades antigas e para
      passivas/transformações.
      Parâmetros: (player, target, damage)
      Para "onBlock"/"onReceiveDamage", o parâmetro 'target' será o atacante.
      Deve usar as funções auxiliares e escalar com
      getNivelPorAfinidade(player:getDominioAfinidade(DOMINIO_ID)).
      NUNCA usar getArmor() em criaturas.

================================================================================
🔧 CAMPOS OPCIONAIS
================================================================================

buffType
  [OP] Tipo de buff automático (em vez de custom):
       "speedBurst"          – acelera velocidade de ataque.
       "nextAttackCondition" – aplica condição no próximo ataque.
       "damageBoost"         – aumenta dano por N ataques.
       "lifeLeech"           – roubo de vida por duração.
       "custom"              – efeito totalmente personalizado (usa 'efeito').
       ⚠️ OBSOLETO para novas habilidades modulares – usar 'efeitoConfig'.

buffMultiplier / buffDurationMs
  [OP] Parâmetros para speedBurst.

conditionType / conditionDuration / conditionDamage
  [OP] Parâmetros para nextAttackCondition.

percentExtra / buffHits
  [OP] Parâmetros para damageBoost.

leechPercent / durationSec
  [OP] Parâmetros para lifeLeech.

condicao
  [OP] Condição especial para activar:
       "cercado"         – 3+ inimigos ao redor do jogador (raio 3).
       "cercadoAlvo"     – 2+ inimigos perto do alvo (raio 2).
       "vidaBaixa"       – alvo com menos de 20% de vida.
       "distancia4"      – alvo a 4+ tiles de distância.
       "distanciaMinima2"– alvo a pelo menos 2 tiles.
       "distanciaMaxima3"– alvo a no máximo 3 tiles.
       "5Golpes"         – 5 ataques consecutivos com a mesma arma.
       "inimigosProximos"– pelo menos 1 inimigo num raio de 2 tiles do jogador.
       "alvoIsolado"     – alvo sem outros inimigos num raio de 2 tiles.
       "aposEsquiva"     – activada após o jogador esquivar.

condicaoFocoMin
  [OP] Condição mínima de foco (0‑100). A habilidade só pode ser usada se a
       barra de foco do domínio principal estiver ≥ este valor.
       Exemplo: 75 = só com a barra ≥ 75% (quase a formar Vínculo).
       Se omitido, a habilidade pode ser usada com qualquer nível de foco.

chiCusto
  [OP] Custo de Chi. Só para habilidades do caminho Artes Marciais.

efeitoVisual
  [OP] Função para efeitos visuais (projécteis, magias no alvo).
       Executada sempre que a habilidade é activada (excepto se
       buffType="custom", onde a própria função efeito já trata).
       Suprimida se já houver mais de 1 execução no mesmo ciclo (anti‑lag).

posturaMod
  [OP] Modificadores por postura.
       chanceMult: multiplicador da prioridade na postura.
       Exemplo: { [1] = { chanceMult = 1.4 }, [2] = { chanceMult = 1.0 },
                  [3] = { chanceMult = 0.6 } }

async
  [OP] Se true, o lock (STORAGE.HABILIDADE_LOCK) é gerido manualmente
       na função efeito. Útil para addEvent assíncronos.

sinergiaDominios
  [OP] Lista de domínios adicionais exigidos para sinergia.
       Exemplo: {23, 24, 25, 200} para uma tempestade elemental que escala
       com todas as especialidades de Elementos.

efeitosSecundarios
  [OP] Tabela de efeitos secundários indexados por domínio.
       Cada entrada é uma função(player, target, damage) chamada se o
       jogador tiver afinidade suficiente (ex.: nível ≥ 10) no domínio.

desfazer
  [OP] Função para desfazer o efeito de uma transformação.
       Chamada por atualizarTransformacoes quando o estado de alma termina.
       Parâmetros: (player)

================================================================================
📦 PACOTES TEMÁTICOS (CATÁLOGO)
================================================================================

Cada habilidade pertence a um ou mais pacotes. Os pacotes definem a mecânica
e o feedback visual da habilidade.

1. Dano em Área (Ground)
   - Explosão centrada num ponto fixo (chão sob o alvo ou o jogador).
   - Efeitos visuais: CONST_ME_GROUNDSHAKER, CONST_ME_FIREAREA, etc.
   - Por omissão, não cria fields.

2. Dano em Área (Alvo)
   - Explosão centrada numa criatura alvo. Acompanha o alvo.
   - Efeitos visuais: CONST_ME_FIREAREA, CONST_ME_ENERGYAREA, etc.

3. Ricochete
   - Projétil que salta do alvo principal para secundários.
   - Efeitos visuais: projétil (CONST_ANI_*) + sendDistanceEffect entre alvos.

4. Corrente / Ligação
   - Liga dois ou mais inimigos. Dano replicado ou puxão.
   - Efeitos visuais: CONST_ME_ENERGYHIT entre os alvos ligados.

5. Knockback / Deslocamento
   - Empurra, puxa ou arremessa o alvo.
   - Efeitos visuais: CONST_ME_HITAREA + empurrarAlvo().

6. Condição (Debuff)
   - Aplica condição negativa (sangramento, atordoamento, paralisia, veneno,
     medo, queimadura).
   - Efeitos visuais: CONST_ME_DRAWBLOOD, CONST_ME_STUN, etc.

7. Buff Pessoal
   - Aumenta velocidade, dano, regeneração, ou concede furtividade.
   - Efeitos visuais: CONST_ME_MAGIC_RED, CONST_ME_MAGIC_BLUE, CONST_ME_POFF.

8. Defesa / Contra‑ataque
   - Bloqueia e retalia, reduz dano recebido, recupera vida/mana.
   - Efeitos visuais: CONST_ME_HITAREA, CONST_ME_MAGIC_BLUE.

9. Transformação
   - Altera o estado do jogador durante Lampejo, Vínculo ou Maestria.
   - Efeitos visuais: CONST_ME_MAGIC_RED, CONST_ME_MAGIC_BLUE.

10. Execução (Finisher)
    - Dano massivo contra alvos com vida baixa (<20%).
    - Efeitos visuais: CONST_ME_DRAWBLOOD, CONST_ME_MORTAREA.

11. Sinergia Elemental
    - Habilidade que escala com o elemento da arma (elementalBond).
    - Já implementado via armaCompativeis no executor.

12. Sinergia de Domínios
    - Habilidade que verifica afinidade em múltiplos domínios.
    - Usar campo sinergiaDominios e efeitosSecundarios.

13. Fields (Campos Elementais)
    - Criam campos no chão que causam dano/efeitos a cada tick.
    - Intensidade escala com o nível de afinidade do conjurador.
    - O conjurador é sempre imune. Com Maestria elemental (nível 20),
      o grupo também fica imune.
    - Tipos de field:
      | Elemento      | Domínio | Efeito do Field                |
      |---------------|---------|--------------------------------|
      | Fogo          | 23      | Queimadura                     |
      | Energia       | 26      | Mini‑paralisia                 |
      | Gelo          | 24      | Lentidão                       |
      | Água          | 24      | Redução de velocidade de ataque|
      | Terra         | 25      | Knockdown (empurrão)           |
      | Veneno        | 25      | Veneno                         |
      | Sagrado       | 200     | Cura aliados, dano a mortos‑vivos|
      | Morte         | 200     | Dreno de vida                  |

📋 CONDIÇÕES DE ACTIVAÇÃO DISPONÍVEIS
================================================================================

"cercado"         – 3+ inimigos ao redor do jogador (raio 3).
"cercadoAlvo"     – 2+ inimigos perto do alvo (raio 2).
"vidaBaixa"       – alvo com menos de 20% de vida.
"distancia4"      – alvo a 4+ tiles de distância.
"distanciaMinima2"– alvo a pelo menos 2 tiles.
"distanciaMaxima3"– alvo a no máximo 3 tiles.
"5Golpes"         – 5 ataques consecutivos com a mesma arma.
"inimigosProximos"– pelo menos 1 inimigo num raio de 2 tiles do jogador.
"alvoIsolado"     – alvo sem outros inimigos num raio de 2 tiles.
"aposEsquiva"     – activada após o jogador esquivar.

Regras de atribuição:
- Habilidades ofensivas ranged NUNCA usam "cercado".
- Habilidades ofensivas corpo‑a‑corpo podem usar "cercado" se forem de área.
- Habilidades defensivas usam "cercado", "inimigosProximos", ou "vidaBaixa"
  (do jogador, verificada no efeito).

================================================================================
🧩 FUNÇÕES AUXILIARES DISPONÍVEIS (funcoes_habilidade.lua v2.0.2)
================================================================================

dispararProjetil(player, target, danoBase, extraPercent)
  Dispara um projétil elemental com dano proporcional.

dispararRajada(player, target, danoBase, extraPercent, numProjeteis, intervaloMs)
  Dispara N projéteis em sequência (intervalo mínimo 250ms).

explosaoElementalArea(player, target, danoBase, extraPercent, raio)
  Explosão elemental ao redor do alvo.

danoElemental(player, target, dano, efeitoOpcional)
  Aplica dano com o elemento da arma.

golpeCorpoACorpo(player, target, dano, efeito)
  Aplica dano físico com efeito visual no alvo (melee).

empurrarAlvo(player, target)
  Empurra o alvo na direção oposta.

criarField(player, posicao, elemento, danoPorTick, duracaoMs, intervaloMs, efeitoVisual)
  Cria um campo elemental no chão.

afetarArea(player, posicao, raio, danoBase, extraPercentDano, curaBase, extraPercentCura, curarAliados)
  Dano a inimigos e cura a aliados numa área.

ligarAlvos(player, alvo1, alvo2, danoBase, extraPercent, efeitoVisual)
  Liga dois alvos com efeito visual e dano.

aplicarCondicaoArea(player, posicao, raio, condType, duration, damagePerTick)
  Aplica condição a todos os inimigos numa área.

getElementalProjectile(player)
  Retorna projétil visual, efeito mágico e tipo de dano com base na arma
  e munição equipadas.

================================================================================
📐 REGRAS GERAIS PARA TODAS AS HABILIDADES
================================================================================

1. ESCALONAMENTO COM O DOMÍNIO:
   Toda habilidade deve escalar com o nível do domínio. Usa
   getNivelPorAfinidade() para obter um multiplicador (0 a 20) e aplica
   nas fórmulas de dano, duração, etc.

2. INFLUÊNCIA DA POSTURA:
   O comportamento pode mudar por postura. As mensagens já são separadas
   por postura.

3. HABILIDADES APRENDIDAS ANTES DO NÍVEL:
   Se desbloqueada por quest/NPC antes do nível mínimo, o efeito usa
   valores base (nível 1).

4. ÁRVORE I‑IV OBRIGATÓRIA (para especialidades):
   I   (nível 5):  Passiva – define a identidade do estilo.
   II  (nível 10): Passiva – expande a primeira.
   III (nível 15): Gatilho ou Transformação – a assinatura do estilo.
   IV  (nível 20): Passiva – Maestria. Concede Serenidade, aperfeiçoa a III
                   e ensina a III ao domínio irmão.

5. ENSINO ENTRE DOMÍNIOS IRMÃOS:
   Na passiva IV, ensinar a habilidade III ao domínio irmão usando
   aprenderHabilidade(p, HAB_IRMA_III).

6. PROJÉTEIS (HABILIDADES DE DISTÂNCIA):
   Usa getElementalProjectile(player) para obter projétil, efeito e elemento.
   Para múltiplos projéteis, addEvent com intervalo >= 250ms.

7. HABILIDADES CORPO‑A‑CORPO:
   Usa golpeCorpoACorpo(player, target, dano, efeito). Para múltiplos golpes,
   addEvent com intervalo. NUNCA usar sendDistanceEffect para corpo‑a‑corpo
   a menos que seja um projétil real.

8. FEEDBACK NARRATIVO (PADRÃO OFICIAL v3.3):
   Ver secção própria acima.

9. BUFFS E DEBUFFS AUTOMÁTICOS:
   Habilidades com buffType = "speedBurst", "damageBoost", "lifeLeech"
   NÃO devem ter 'mensagem' (as funções de buff já enviam feedback).

10. CHI:
    Habilidades com chiCusto > 0 são automaticamente validadas pelo executor.

11. NUNCA USAR getArmor() EM CRIATURAS:
    Usar dano percentual extra baseado no nível do domínio.

12. PASSIVAS AUTO‑SUFICIENTES:
    O campo 'efeito' de uma passiva deve aplicar directamente o bónus real:
    - Usar Condition(CONDITION_ATTRIBUTES, ...) para atributos permanentes.
    - Usar player:setCombatModifier("chave", valor) para modificadores.
    - NUNCA apenas escrever storages que dependam de módulo externo.

13. TRANSFORMAÇÕES COM DESFAZER:
    O campo 'efeito' aplica o bónus. O campo 'desfazer' (opcional) reverte‑o.

14. FIELDS (CAMPOS ELEMENTAIS):
    - Apenas habilidades específicas (geralmente ligadas a transformações
      de Maestria) criam fields.
    - O conjurador é SEMPRE imune aos seus próprios fields.
    - Com a transformação de Maestria adequada, o grupo também fica imune.
