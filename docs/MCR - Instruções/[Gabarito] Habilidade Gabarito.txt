PROJETO MCR – HABILIDADE GABARITO (Documento Canónico)
Versão 4.7 – Sistema de Progressão do Aventureiro (SPA v4.6)
Arquivo: [Gabarito] Habilidade Gabarito.txt

🎯 OBJECTIVO DESTE DOCUMENTO
Este documento é a referência máxima para a criação de qualquer habilidade
no Projeto MCR. Define todos os campos possíveis, as regras de feedback
narrativo, os pacotes temáticos, as directrizes de implementação e as
convenções que garantem a coerência e a qualidade do SPA.

O SPA adopta uma arquitectura modular em três camadas:

- Motor de Habilidades (motor_habilidades.lua v2.0) – 24 tipos de efeitos
  padronizados, fallback automático de efeitos visuais, suporte a
  sinergias escalonadas e efeitos temporais (storm, orbit, rain, pulse,
  trap).
- Definições de Habilidades (dados/*.lua) – apenas parâmetros, zero lógica.
- Narrativa (narrativa_habilidades.lua) – textos de feedback automáticos.

A versão 4.7 consolida a remoção total do sistema legado buffType e
documenta os novos tipos de efeito e o sistema de contenção por categoria
(limites de execução por tipo de habilidade).

Sempre que um criador de conteúdo ou assistente for gerar uma nova
habilidade, DEVE seguir este gabarito.

================================================================================
📦 ESTRUTURA GERAL DE UMA HABILIDADE
================================================================================

Cada habilidade é uma entrada na tabela global HABILIDADES[habId] = { ... }.
Os campos estão organizados em:

[O]  = Obrigatório
[C]  = Condicional (obrigatório apenas para certos tipos)
[OP] = Opcional

================================================================================
🧩 ARQUITECTURA MODULAR (ACTUALIZADA v2.0)
================================================================================

Todas as habilidades de gatilho são definidas através do campo 'efeitoConfig',
uma tabela de parâmetros interpretada pelo motor central (motor_habilidades.lua
v2.0). O motor também fornece:

- Fallback automático de efeitos visuais baseado no elemento da arma equipada
  (tabela EFEITOS_ELEMENTAIS). O campo 'magicEffect' é OPCIONAL.
- Adaptação elemental para Wands (210) e Rods (211) – o efeito visual e a
  condição são automaticamente ajustados ao elemento da varinha.
- Suporte a 'conditionMagicEffect' para separar o efeito visual da condição
  do efeito de dano.
- Novos tipos modulares: sinergia_escalonada, storm, orbit, rain, pulse, trap.
- Sistema de contenção por categoria no executor: cada categoria (aoe, single,
  debuff, etc.) tem um número máximo de execuções a cada 10 segundos.

Vantagens:
- Novas habilidades são criadas apenas com parâmetros, sem código repetitivo.
- O feedback narrativo é gerado automaticamente pela Narrativa.
- Alterações no motor propagam-se para todas as habilidades existentes.
- O sistema antigo buffType foi completamente removido; não é mais suportado.
- Performance controlada: limites de categoria evitam spam da mesma habilidade.

================================================================================
🏷️ CAMPOS OBRIGATÓRIOS
================================================================================

nome
  [O] Nome da habilidade. Aparece como emote sobre o jogador
      (TALKTYPE_MONSTER_SAY) e nos menus de diagnóstico.
  Exemplo: "Investida Brutal"

tipo
  [O] Tipo da habilidade:
      "passiva"        – efeito permanente, aplicado no login e renovado
                         a cada 2 segundos pelo think_master.
      "gatilho"        – ativada por eventos: onAttack, onDealtDamage,
                         onKill, onBlock, onReceiveDamage.
                         Usa OBRIGATORIAMENTE o campo 'efeitoConfig'.
      "transformacao"  – activa/desactiva com base nos Estados de Alma
                         (Lampejo/Vínculo/Maestria). Suporta o campo
                         'desfazer' para reverter os bónus.

dominio
  [O] Lista de IDs de domínios onde esta habilidade se aplica.
      Exemplo: {111} = só Machados Pesados; {110, 111} = Machados Leves
      e Pesados.

nivelMin
  [O] Nível mínimo do domínio principal para aprender a habilidade
      automaticamente. Se aprendida antes (por quest/NPC), funciona
      mas com valores base (nível 1).

descricaoEfeito
  [O] Descrição curta do efeito. Aparece APENAS ao aprender a habilidade
      (chat colorido). Pode ser string fixa ou função(player) que retorna
      uma string.

cor
  [O] Cor base da habilidade, usada no emote, nos colchetes da mensagem
      e na mensagem de aprendizagem. Usar SEMPRE a cor do domínio
      principal (COR.DOM_COMBATE_* ou COR.DOM_MAGIA_*).

mensagem
  [O] Mensagens de feedback narrativo (canal 24).
      Com a arquitectura modular, este campo é OPCIONAL para
      habilidades de gatilho que usam 'efeitoConfig' – a Narrativa
      gera o feedback automaticamente.
      Se quiseres mensagens personalizadas, aplicam-se as mesmas regras:
      Para cada postura [1]=Ímpeto, [2]=Equilíbrio, [3]=Guarda:
        Pode ser uma tabela de funções (player, target) que retornam a
        string formatada.
        Pode ser uma única função (player, target).
        Pode ser uma tabela de strings estáticas (para passivas e
        transformações).
      Se não houver entrada para uma postura, o executor usa [2].

efeitoConfig
  [O para gatilhos modulares] Tabela de parâmetros que o motor central
      interpreta para executar o efeito.
      Deve conter obrigatoriamente o campo 'tipo', que define qual efeito
      será aplicado. Os restantes campos dependem do tipo escolhido.
      O campo 'magicEffect' é OPCIONAL – se omitido, o motor usará um
      efeito visual adequado ao elemento da arma equipada (fallback
      automático).
      Ver secção "TIPOS DE EFEITOS DO MOTOR" abaixo.

efeito
  [O para passivas e transformações; obsoleto para gatilhos]
      Função que executa o efeito da passiva ou transformação.
      Para passivas que afectam vida, mana ou velocidade de movimento,
      o campo deve ficar vazio (função vazia) – a gestão é feita
      centralmente por recalcularVidaMana() e recalcularVelocidade().
      Para outras passivas, usar setCombatModifier ou Conditions
      auto‑suficientes (CONDITION_ATTRIBUTES).
      Parâmetros: (player, target, damage)
      Para "onBlock"/"onReceiveDamage", o parâmetro 'target' será o atacante.
      NUNCA usar getArmor() em criaturas.

================================================================================
💬 REGRAS DE FEEDBACK LIMPO (v3.3)
================================================================================

FORMATOS POR TIPO DE HABILIDADE:

  • Ataque SEM efeito adicional:
    "[Nome] Frase narrativa normal com o Alvo colorido."

  • Ataque COM efeito (sangramento, atordoamento, knockback, veneno, etc.):
    "[Nome] Frase curta em Alvo - Efeito."

  • Defesa (onBlock / onReceiveDamage):
    "[Nome] Frase narrativa sobre o bloqueio/defesa contra Alvo - Efeito."

  • Múltiplos efeitos:
    "[Nome] Frase narrativa em Alvo - Efeito1, Efeito2 e Efeito3."

REGRAS ABSOLUTAS:
  - Começar SEMPRE com c("[Nome da Habilidade] ", corDoDominio).
  - Usar c(t:getName(), COR.INIMIGO_MONSTRO) para o nome do alvo.
  - Apenas o NOME DO ALVO e 1‑2 EFEITOS (condições, acções) são coloridos.
  - O texto narrativo é NEUTRO (cor normal do chat).
  - NUNCA usar travessões longos (—). Usar hífen (-).
  - NUNCA colorir palavras descritivas como "lâmina", "sangue", "impacto".
    Apenas termos de efeitos mecânicos: "Sangramento", "Atordoado", etc.
  - Cores de efeitos: COR.COND_* para condições, COR.ACAO_* para acções
    físicas, COR.SUCESSO_CURA para curas, COR.DANO_CRITICO para dano massivo.

================================================================================
⚙️ TIPOS DE EFEITOS DO MOTOR (efeitoConfig.tipo)
================================================================================

Cada habilidade de gatilho que usa o motor modular deve definir um
campo 'efeitoConfig' com um 'tipo' da lista abaixo.

NOTA: O campo 'magicEffect' é OPCIONAL em todos os tipos que o possuem.
Se omitido, o motor obtém automaticamente o efeito visual a partir do
elemento da arma equipada (tabela EFEITOS_ELEMENTAIS). O campo
'conditionMagicEffect' também é opcional e controla o efeito visual
específico quando uma condição é aplicada.

═══════════════════════════════════════════════════════════════
TIPOS CLÁSSICOS (ATAQUES E EFEITOS DIRECTOS)
═══════════════════════════════════════════════════════════════

dano_extra
  - Dano adicional simples (corpo a corpo ou projétil).
  - Campos: percentual, tipo ("distancia"|nil), magicEffect (opcional)
  - Exemplo: { tipo = "dano_extra", percentual = 0.25 }

area_ground
  - Explosão circular no chão com onda expansiva.
  - Campos: percentual, centro ("jogador"|"alvo"), raio, magicEffect (opcional),
    knockback (bool), condicao (CONDITION_*), condDuration, periodicDamage,
    conditionMagicEffect (opcional)
  - A onda expansiva é controlada por throttle (máximo 1 animação completa
    a cada 2 segundos; as restantes usam efeito simplificado).
  - Exemplo: { tipo = "area_ground", centro = "jogador", raio = 1, percentual = 0.4 }

area_target
  - Explosão circular centrada no alvo, com onda expansiva.
  - Campos: percentual, areaPercentual, raio, magicEffect (opcional),
    knockback (bool)
  - Exemplo: { tipo = "area_target", percentual = 0.5, areaPercentual = 0.3, raio = 2 }

ricochete
  - Projétil que salta do alvo principal para N inimigos próximos.
  - Campos: percentual, raio, saltos, magicEffect (opcional)
  - Exemplo: { tipo = "ricochete", percentual = 0.4, raio = 2, saltos = 2 }

knockback
  - Empurra o alvo. Pode ou não causar dano adicional.
  - Campos: comDano (bool), percentual, magicEffect (opcional)
  - Exemplo: { tipo = "knockback", comDano = true, percentual = 0.3 }

condicao
  - Aplica uma condição ao alvo, com ou sem dano adicional.
  - Campos: conditionType, duration, periodicDamage, comDano (bool),
    percentual, magicEffect (opcional), conditionMagicEffect (opcional)
  - Exemplo: { tipo = "condicao", conditionType = CONDITION_BLEEDING,
    duration = 4000, periodicDamage = 15, conditionMagicEffect = CONST_ME_DRAWBLOOD }

rajada
  - Dispara N projéteis em sequência contra o alvo.
  - Campos: percentual, numProjeteis, intervaloMs
  - Exemplo: { tipo = "rajada", percentual = 0.3, numProjeteis = 3, intervaloMs = 250 }

corrente
  - Liga dois alvos, causando dano ao secundário.
  - Campos: percentual, raio, magicEffect (opcional)
  - Exemplo: { tipo = "corrente", percentual = 0.4, raio = 2 }

finisher
  - Dano massivo contra alvos com vida baixa (<20%).
  - Campos: percentual, magicEffect (opcional), knockback (bool),
    medoArea (bool), sangramentoArea (bool)
  - Exemplo: { tipo = "finisher", percentual = 1.5, medoArea = true }

═══════════════════════════════════════════════════════════════
TIPOS DE BUFF E LEECH
═══════════════════════════════════════════════════════════════

buff_speed
  - Aceleração de ataque (velocidade de ataque, não movimento).
  - Campos: multiplier, durationMs
  - Exemplo: { tipo = "buff_speed", multiplier = 0.4, durationMs = 700 }

buff_damage
  - Aumento de dano por N ataques.
  - Campos: percentExtra, buffHits
  - Exemplo: { tipo = "buff_damage", percentExtra = 20, buffHits = 3 }

life_leech
  - Roubo de vida por duração.
  - Campos: leechPercent, durationSec
  - Exemplo: { tipo = "life_leech", leechPercent = 15, durationSec = 4 }

mana_leech
  - Roubo de mana por duração.
  - Campos: leechPercent, durationSec
  - Exemplo: { tipo = "mana_leech", leechPercent = 10, durationSec = 4 }

next_condition
  - Prepara uma condição para o próximo ataque.
  - Campos: conditionType, duration, periodicDamage
  - Exemplo: { tipo = "next_condition", conditionType = CONDITION_BLEEDING,
    duration = 3500, periodicDamage = 12 }

═══════════════════════════════════════════════════════════════
TIPOS DE DEFESA
═══════════════════════════════════════════════════════════════

defesa_cura
  - Recupera vida ao bloquear.
  - Campos: percentual, reflect (bool), reflectPercentual
  - Exemplo: { tipo = "defesa_cura", percentual = 0.04, reflect = true,
    reflectPercentual = 0.2 }

defesa_barreira
  - Reduz dano recebido temporariamente.
  - Campos: factor, durationMs
  - Exemplo: { tipo = "defesa_barreira", factor = 0.5, durationMs = 2000 }

defesa_contra_ataque
  - Contra-ataca todos os inimigos ao redor ao bloquear.
  - Campos: percentual, magicEffect (opcional)
  - Exemplo: { tipo = "defesa_contra_ataque", percentual = 0.3 }

═══════════════════════════════════════════════════════════════
TIPOS DE ÁREA PERSISTENTE
═══════════════════════════════════════════════════════════════

field
  - Cria um campo elemental no chão que aplica dano sequencial.
  - Campos: centro ("jogador"|"alvo"), elemento, danoPorTick, duracaoMs,
    intervaloMs, magicEffect (opcional), raio (opcional, padrão 0)
  - O dano é aplicado sequencialmente (300ms entre alvos) para evitar lag.
  - O conjurador é SEMPRE imune aos seus próprios campos.
  - Exemplo: { tipo = "field", centro = "alvo", elemento = COMBAT_FIREDAMAGE,
    danoPorTick = 10, duracaoMs = 5000, raio = 1 }

═══════════════════════════════════════════════════════════════
TIPOS DE SINERGIA
═══════════════════════════════════════════════════════════════

sinergia
  - Combina múltiplos domínios com efeitos manuais.
  - Usa os campos 'sinergiaDominios' e 'efeitosSecundarios' da habilidade.
  - Campos: efeitoBase (tipo de efeito a aplicar primeiro), nivelMinSecundario
  - Exemplo: { tipo = "sinergia", efeitoBase = "area_target", nivelMinSecundario = 10 }

sinergia_escalonada (NOVO)
  - Executa efeitos que escalam automaticamente com o nível de afinidade.
  - Campos:
    efeitoBase – tipo do efeito base (opcional)
    efeitoBaseConfig – parâmetros do efeito base
    efeitosPorDominio – tabela { [domId] = { nivelMin, efeitoConfig, escala (opcional), chance (opcional), escalaChance (opcional) } }
    efeitoEpico – { efeitoConfig } executado se TODOS os domínios em efeitosPorDominio estiverem em nível 20
  - A tabela 'escala' mapeia parâmetros do efeitoConfig para multiplicadores de crescimento por nível.
  - O campo 'chance' (0-100) define uma probabilidade base que aumenta com 'escalaChance' por nível.
  - Exemplo: ver secção de exemplos no final do documento.

═══════════════════════════════════════════════════════════════
TIPOS TEMPORAIS (EFEITOS PERSISTENTES NO TEMPO) – NOVOS
═══════════════════════════════════════════════════════════════

Estes efeitos criam eventos repetidos ao longo do tempo. O executor
impõe um limite de 2 efeitos temporais activos por jogador para
garantir performance (MAX_TEMP_EFFECTS = 2).

storm
  - Tempestade: atinge um inimigo aleatório na área a cada intervaloMs.
  - Campos: centro ("jogador"|"alvo"), raio, elemento, danoPorTick,
    intervaloMs, duracaoMs, magicEffect (opcional)
  - Exemplo: { tipo = "storm", centro = "alvo", raio = 3,
    elemento = COMBAT_ENERGYDAMAGE, danoPorTick = 25, intervaloMs = 450,
    duracaoMs = 3600 }

orbit
  - Órbita: esferas que orbitam o jogador e causam dano a inimigos próximos.
  - Campos: numEsferas, raio, elemento, danoPorTick, intervaloMs,
    duracaoMs, magicEffect (opcional)
  - Exemplo: { tipo = "orbit", numEsferas = 3, raio = 2,
    elemento = COMBAT_FIREDAMAGE, danoPorTick = 15, intervaloMs = 600,
    duracaoMs = 5000 }

rain
  - Chuva: projéteis caem aleatoriamente na área em intervalos regulares.
  - Campos: centro ("jogador"|"alvo"), raio, elemento, danoPorTick,
    intervaloMs, duracaoMs, magicEffect (opcional)
  - Exemplo: { tipo = "rain", centro = "alvo", raio = 3,
    elemento = COMBAT_FIREDAMAGE, danoPorTick = 20, intervaloMs = 300,
    duracaoMs = 4000 }

pulse
  - Pulso: ondas de energia que se expandem periodicamente.
  - Campos: raioMax, elemento, danoPorPulso, intervaloMs, numPulsos,
    magicEffect (opcional)
  - Exemplo: { tipo = "pulse", raioMax = 3, elemento = COMBAT_HOLYDAMAGE,
    danoPorPulso = 30, intervaloMs = 800, numPulsos = 5 }

trap
  - Armadilha tática: marca no chão que detona quando um inimigo pisa.
  - Campos: centro ("alvo"|"jogador"|"entre"|"fuga"|"aleatorio"),
    duracaoMs, primeDelayMs (tempo até armar), elemento, dano, raio,
    magicEffect (explosão), markEffect (marcador visual),
    condicao (opcional), condDuration (opcional), periodicDamage (opcional)
  - Centros especiais:
    "entre" – a meio caminho entre jogador e alvo (defensivo)
    "fuga" – atrás do alvo, na direcção oposta ao jogador (corta retirada)
    "aleatorio" – posição aleatória ao redor do alvo
  - Exemplo: { tipo = "trap", centro = "entre", duracaoMs = 8000,
    primeDelayMs = 1000, elemento = COMBAT_ENERGYDAMAGE, dano = 60, raio = 2,
    markEffect = CONST_ME_MAGIC_BLUE, condicao = CONDITION_PARALYZE,
    condDuration = 2500 }

================================================================================
🎯 CAMPOS CONDICIONAIS (gatilho)
================================================================================

gatilho
  [C] Gatilho que activa a habilidade:
      "onAttack"         – ao causar dano.
      "onDealtDamage"    – após causar dano.
      "onKill"           – ao matar uma criatura.
      "onBlock"          – ao bloquear um ataque (exige escudo).
      "onReceiveDamage"  – ao receber dano (defesa genérica, não exige arma).

cooldown
  [C] Cooldown individual em segundos.

prioridade
  [C] Prioridade base (1‑10). Habilidades com maior prioridade são
      escolhidas primeiro pelo motor inteligente.

categoria
  [C] Categoria da habilidade. Usada pelo sistema de limites de execução
      e pelos scores contextuais:
      "single"       – dano directo elevado (limite: 3 a cada 10s).
      "aoe"          – atinge múltiplos alvos (limite: 2 a cada 10s).
      "debuff"       – aplica condição negativa (limite: 2 a cada 10s).
      "buff"         – melhora o próprio jogador (limite: 2 a cada 10s).
      "defense"      – cura, escudo, redução de dano (limite: 2 a cada 10s).
      "chi_spender"  – consome Chi (requer chiCusto definido).
      "finisher"     – efeito extra quando alvo está com vida baixa (limite: 1 a cada 10s).
      "sinergia"     – habilidade que escala com múltiplos domínios (limite: 2 a cada 10s).
      "condicao"     – especializada em aplicar uma condição prolongada.

      Se a categoria não for definida, o sistema assume "single" como
      fallback. Isto garante que mesmo habilidades antigas sejam limitadas.

================================================================================
🔧 CAMPOS OPCIONAIS
================================================================================

⚠️ buffType e todos os campos associados foram removidos do executor.
Não devem ser usados em novas habilidades.

condicao
  [OP] Condição especial para activar:
       "cercado"         – 3+ inimigos ao redor do jogador (raio 3).
       "cercadoAlvo"     – 2+ inimigos perto do alvo (raio 2).
       "vidaBaixa"       – alvo com menos de 20% de vida.
       "distancia4"      – alvo a 4+ tiles de distância.
       "distanciaMinima2"– alvo a pelo menos 2 tiles.
       "distanciaMaxima3"– alvo a no máximo 3 tiles.
       "5Golpes"         – 5 ataques consecutivos com a mesma arma.
       "inimigosProximos"– pelo menos 1 inimigo num raio de 2 tiles do jogador.
       "alvoIsolado"     – alvo sem outros inimigos num raio de 2 tiles.
       "aposEsquiva"     – activada após o jogador esquivar.
       "alvoProximo"     – alvo a no máximo X tiles (usar com condicaoDist).

condicaoDist
  [OP] Distância máxima para a condição "alvoProximo" (padrão: 3).

condicaoFocoMin
  [OP] Condição mínima de foco (0‑100). A habilidade só pode ser usada se a
       barra de foco do domínio principal estiver ≥ este valor.
       Distribuição típica: básicas 0%, condições 25%, área 50%,
       knockback 75%, finisher 90%.

conditionMagicEffect
  [OP] Efeito visual quando uma condição é aplicada ao alvo.
       Aplica‑se aos tipos "condicao", "area_ground", "area_target",
       "storm" (se tiver condicao), "trap" (se tiver condicao).
       Se omitido, o motor usa o valor de 'magicEffect' (ou o fallback elemental).
  Exemplo: conditionMagicEffect = CONST_ME_GREEN_RINGS

escala
  [OP] Usado exclusivamente em sinergia_escalonada. Tabela que mapeia
       parâmetros do efeitoConfig para multiplicadores de crescimento.
       Exemplo: escala = { duration = 200, periodicDamage = 1 }
       → a cada nível, duration aumenta 200ms e periodicDamage +1.

chance
  [OP] Usado em sinergia_escalonada. Probabilidade base (0‑100) de o
       efeito do domínio ser executado. Exemplo: chance = 25.

escalaChance
  [OP] Usado em sinergia_escalonada. Aumento percentual por nível.
       Exemplo: escalaChance = 1 → +1% de chance por nível.

chiCusto
  [OP] Custo de Chi. Só para habilidades do caminho Artes Marciais.

efeitoVisual
  [OP] Função para efeitos visuais (projécteis, magias no alvo).
       Suprimida se já houver mais de 1 execução no mesmo ciclo (anti‑lag).

posturaMod
  [OP] Modificadores por postura.
       chanceMult: multiplicador da prioridade na postura.
       Exemplo: { [1] = { chanceMult = 1.4 }, [2] = { chanceMult = 1.0 },
                  [3] = { chanceMult = 0.6 } }

async
  [OP] Se true, o lock é gerido manualmente na função efeito.

sinergiaDominios
  [OP] Lista de domínios adicionais exigidos para sinergia.

efeitosSecundarios
  [OP] Tabela de efeitos secundários indexados por domínio (funções manuais).
       Preterido em favor de sinergia_escalonada para novas habilidades.

desfazer
  [OP] Função para desfazer o efeito de uma transformação.

markEffect
  [OP] Usado exclusivamente em "trap". Efeito visual do marcador enquanto
       a armadilha está armada. Exemplo: markEffect = CONST_ME_MAGIC_BLUE

primeDelayMs
  [OP] Usado em "trap". Tempo em ms até a armadilha armar (padrão: 1000).

================================================================================
📦 PACOTES TEMÁTICOS (CATÁLOGO)
================================================================================

1. Dano em Área (Ground) – Explosão centrada no chão.
2. Dano em Área (Alvo) – Explosão centrada numa criatura.
3. Ricochete – Projétil que salta entre inimigos.
4. Corrente / Ligação – Liga dois ou mais inimigos.
5. Knockback / Deslocamento – Empurra, puxa ou arremessa.
6. Condição (Debuff) – Aplica condição negativa.
7. Buff Pessoal – Aumenta velocidade, dano, regeneração.
8. Defesa / Contra‑ataque – Bloqueia e retalia.
9. Transformação – Altera o estado do jogador.
10. Execução (Finisher) – Dano massivo contra alvos com vida baixa.
11. Sinergia Elemental – Escala com o elemento da arma.
12. Sinergia de Domínios – Escala com afinidade em múltiplos domínios.
13. Fields (Campos Elementais) – Dano contínuo no chão.
14. Movimento – Velocidade de deslocamento (passivas centrais).
15. Tempestade (storm) – Dano aleatório em área ao longo do tempo.
16. Órbita (orbit) – Esferas que orbitam o jogador.
17. Chuva (rain) – Projéteis caindo aleatoriamente em área.
18. Pulso (pulse) – Ondas de dano expansivas periódicas.
19. Armadilha (trap) – Marca no chão que detona reactivamente.
20. Sinergia Escalonada – Efeitos que crescem com o nível do domínio.

================================================================================
📋 CONDIÇÕES DE ACTIVAÇÃO DISPONÍVEIS
================================================================================

"cercado"         – 3+ inimigos ao redor do jogador (raio 3).
"cercadoAlvo"     – 2+ inimigos perto do alvo (raio 2).
"vidaBaixa"       – alvo com menos de 20% de vida.
"distancia4"      – alvo a 4+ tiles de distância.
"distanciaMinima2"– alvo a pelo menos 2 tiles.
"distanciaMaxima3"– alvo a no máximo 3 tiles.
"5Golpes"         – 5 ataques consecutivos com a mesma arma.
"inimigosProximos"– pelo menos 1 inimigo num raio de 2 tiles do jogador.
"alvoIsolado"     – alvo sem outros inimigos num raio de 2 tiles.
"aposEsquiva"     – activada após o jogador esquivar.
"alvoProximo"     – alvo a no máximo X tiles (define X com condicaoDist).

================================================================================
📝 NOTAS FINAIS
================================================================================

- Mantém os ficheiros de habilidades bem comentados e estruturados.
- Usa a faixa de storages exclusiva do domínio.
- Testa cada habilidade com !platinar e !aprendertudo.
- As mensagens devem estar em PT‑BR, imersivas, com acentuação correcta
  (ISO‑8859‑1).
- Ao adicionar novas habilidades, sincroniza com a base de dados via
  init_db.lua.
- Cores seguem a tabela COR v10.0 (texto_colorido.lua).
- Feedback segue o PADRÃO OFICIAL LIMPO v3.3.
- Para habilidades de gatilho, usa SEMPRE 'efeitoConfig'. O campo 'efeito'
  manual está reservado exclusivamente para passivas e transformações.
- Passivas de vida, mana e velocidade devem ter 'efeito' vazio.
- O campo 'magicEffect' é opcional em todos os tipos de efeito.
- O campo 'conditionMagicEffect' permite personalizar o efeito visual
  da aplicação de condições.
- Novas habilidades devem definir SEMPRE o campo 'categoria' para se
  beneficiarem do sistema de limites de execução.
- Habilidades temporais (storm, orbit, rain, pulse, trap) estão limitadas
  a 2 activas por jogador; se o limite for excedido, a habilidade entra
  em cooldown mas não cria o efeito.
