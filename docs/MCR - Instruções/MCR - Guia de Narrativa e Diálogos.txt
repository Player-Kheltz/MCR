🎯 OBJETIVO DESTE GUIA
Este guia é a bússola narrativa do Projeto MCR. Define como os NPCs devem
falar, pensar e reagir dentro do universo de fantasia original "Aventura Fantasiosa",
utilizando a API real do Canary 3.4.1 (Game.createNpcType) e a codificação
ISO-8859-1 obrigatória para scripts que exibem strings ao jogador.

Conteúdo:
- Esqueleto funcional de NPC (nova API).
- Módulo pronouns.lua (sistema nativo de pronomes).
- Biblioteca npc_utils.lua (sinónimos, pronomes, variação).
- Sistema de destaque de palavras com {chaves} (compatível com o OTClient).
- Múltiplas falas com npcHandler:say({...}).
- Regras para criação/modificação de diálogos.
- Personalidade do NPC e progressão de intimidade.
- Uso correto do pronome do personagem (players.pronoun) e do pronome da conta
  (accounts.player_pronoun).

================================================================================
⚠️ REGRAS DE REVISÃO PARA O ASSISTENTE (NARRATIVA E DIÁLOGOS)
================================================================================

Ao criar ou modificar QUALQUER script de NPC (.lua) no servidor, o assistente DEVE:

1. Todo texto exibido ao jogador em PT-BR impecável, com acentuação correta.
   O arquivo DEVE ser salvo como ISO-8859-1 (Latin-1). Strings com acentos
   salvos em UTF-8 PROVOCAM CORRUPÇÃO no cliente (Fase 22.3, verificação do usuário).

2. Usar OBRIGATORIAMENTE a API moderna:
   - Criar o tipo com Game.createNpcType(npcName)
   - Atribuir todos os callbacks de engine (onThink, onAppear, onDisappear,
     onMove, onSay, onCloseChannel) delegando ao npcHandler.
   - Registrar o NPC com npcType:register(npcConfig).
   - Usar a assinatura correta do creatureSayCallback: (npc, player, type, msg).
   - npcHandler:say aceita string única ou tabela de strings com delay.

3. Usar o módulo pronouns.lua para obter pronomes do personagem:
   - Carregar com dofile('data/MCR Scripts/pronouns.lua')
   - Usar Pronouns.getSubject(player), Pronouns.getObject(player), etc.
   - NÃO usar storage 100001 para pronome do personagem.
   - A coluna nativa players.pronoun é preenchida pelo Oráculo e usada
     automaticamente pelo módulo.

4. Respeitar a distinção entre:
   - Pronome da CONTA (jogador/jogadora): usado apenas pelo Oráculo.
     **ATUALIZADO**: O Oráculo pergunta o pronome da conta apenas nos fluxos de
     criação (tópico 5) e gerenciamento (tópico 21), nunca na saudação inicial.
   - Pronome do PERSONAGEM (ele/ela): usado por todos os NPCs e mensagens.

5. Palavras importantes no chat DEVEM ser destacadas com {chaves}, que o
   OTClient Redemption exibe como destaque visual automático (padrão Tibia).
   Exemplos: {aceitar}, {missão}, {Lágrimas de Sereia}.
   NUNCA usar "}?" junto; sempre "} ?" para evitar corrupção visual.

6. Diálogos NÃO devem usar addEvent manual; para sequências de falas,
   utilize a capacidade do handler de receber uma tabela de strings.

7. Toda interação administrativa (loja, banco) pode usar janela modal, mas
   DEVE ser precedida de uma fala narrativa no chat.

8. Incluir prints de debug (prefixo "DEBUG:") em todos os callbacks do NPC,
   especialmente em creatureSayCallback, greetCallback e farewellCallback,
   documentando a entrada de cada bloco condicional, mudanças de tópico e
   decisões tomadas. Os prints devem ser em português, com os valores das
   variáveis relevantes.

9. O NPC deve descobrir e respeitar o pronome do personagem através do
   módulo pronouns.lua, nunca assumindo pelo sexo.

10. A forma como o NPC pergunta ou infere o pronome do personagem DEVE ser
    coerente com a personalidade do NPC. Pode ser directa, subtil, ou
    completamente ausente em NPCs neutros. O objectivo é imersão total.

11. A progressão de intimidade com o NPC deve refletir-se na linguagem:
    um NPC frio pode tornar-se mais caloroso depois de uma quest;
    um NPC desconfiado pode passar a usar elogios. O pronome do personagem
    (uma vez conhecido) influencia essa progressão.

================================================================================
🌌 1. TOM DE VOZ E AMBIENTAÇÃO – O UNIVERSO MCR
================================================================================

O Projeto MCR se passa em um universo de fantasia original. NPCs NÃO são
terminais de missão, mas habitantes vivos. Ao criar um NPC, reflita:
- Qual a história dele dentro do mundo?
- Como ele reage a guerreiros, magos, druidas?
- Qual seu medo, objetivo, mania?
- Como a sua personalidade influencia a maneira como se dirige ao jogador?
  (ex.: um ferreiro rude fala de forma brusca; uma sacerdotisa é calorosa)

================================================================================
📚 2. MÓDULO PRONOUNS.LUA (OBRIGATÓRIO)
================================================================================

Arquivo: data/MCR Scripts/pronouns.lua
Salvar como: ISO-8859-1

Este módulo encapsula o sistema nativo de pronomes do Canary e deve ser
carregado por todos os NPCs que se dirigem ao jogador. Ele oferece funções
simples que automaticamente consultam a coluna players.pronoun e fazem
fallback para o sexo do personagem quando o pronome não está definido.

Funções disponíveis:
- Pronouns.getSubject(player)       → "he", "she", "they"
- Pronouns.getObject(player)        → "him", "her", "them"
- Pronouns.getPossessive(player)    → "his", "her", "their"
- Pronouns.getReflexive(player)     → "himself", "herself", "themself"

Exemplo de uso:
local subject = Pronouns.getSubject(player)
npcHandler:say("Que " .. subject .. " seja bem-vindo" .. (subject == "she" and "a" or "") .. "!", npc, player)

O módulo utiliza as constantes PLAYERPRONOUN_HE, PLAYERPRONOUN_SHE, etc.,
que são expostas pelo servidor.

- O comando `look` do servidor (creaturescripts e eventos) agora chama
  `Pronouns.getSubject(player)` para flexionar a descrição. Exemplo:
  "Você vê um viajante." / "Você vê uma viajante."
  A máscara de Alma também é aplicada para personagens Guest_.

```lua
--[[
Projeto: MCR
NPC: NomeDoNPC (exemplo educativo)
Descrição: NPC de demonstração usando a API real do Canary 3.4.1,
           com pronomes nativos e progressão de intimidade.

dofile('data/MCR Scripts/npc_utils.lua')
dofile('data/MCR Scripts/pronouns.lua')

local npcName = "MeuNPC"
local npcType = Game.createNpcType(npcName)
local npcConfig = {}
npcConfig.name = npcName
npcConfig.description = npcName
npcConfig.health = 100
npcConfig.maxHealth = 100
npcConfig.outfit = {
    lookType = 128, lookHead = 0, lookBody = 0,
    lookLegs = 0, lookFeet = 0, lookAddons = 2
}

local keywordHandler = KeywordHandler:new()
local npcHandler = NpcHandler:new(keywordHandler)

-- Callbacks obrigatórios
npcType.onThink = function(npc, interval) npcHandler:onThink(npc, interval) end
npcType.onAppear = function(npc, creature) npcHandler:onAppear(npc, creature) end
npcType.onDisappear = function(npc, creature) npcHandler:onDisappear(npc, creature) end
npcType.onMove = function(npc, creature, from, to) npcHandler:onMove(npc, creature, from, to) end
npcType.onSay = function(npc, creature, type, msg) npcHandler:onSay(npc, creature, type, msg) end
npcType.onCloseChannel = function(npc, player) npcHandler:onCloseChannel(npc, player) end

local function greetCallback(npc, player)
    local subject = Pronouns.getSubject(player)
    local frase = (subject == "she") and "Bem-vinda!" or "Bem-vindo!"
    npcHandler:setMessage(MESSAGE_GREET, frase)
    return true
end
npcHandler:setCallback(CALLBACK_GREET, greetCallback)

local function creatureSayCallback(npc, player, type, msg)
    if not npcHandler:checkInteraction(npc, player) then return false end
    local subj = Pronouns.getSubject(player)
    if NpcUtils.correspondeAcao(msg, {"missão", "ajuda"}) then
        npcHandler:say("Claro, " .. subj .. "! Aceita uma {missão}?", npc, player)
    end
    return true
end
npcHandler:setCallback(CALLBACK_MESSAGE_DEFAULT, creatureSayCallback)

npcHandler:setMessage(MESSAGE_FAREWELL, "Volte sempre!")
npcHandler:addModule(FocusModule:new(), npcConfig.name, true, true, true)
npcType:register(npcConfig)

🗣️ 6. RECONHECIMENTO FLEXÍVEL DE PALAVRAS-CHAVE
================================================================================

NUNCA faça if msg == "missao". SEMPRE use:
if NpcUtils.correspondeAcao(msg, {"missao", "quest", "tarefa", "trabalho", "ajuda"}) then

A lista de sinónimos deve ser abundante (mínimo 7, idealmente 15+), incluindo
variações regionais e gírias.

Para pronomes do personagem, incluir (adicionar mais se possível):
{"ele", "masculino", "homem", "macho", "menino", "rapaz", "garoto", "m"}
{"ela", "feminino", "mulher", "fêmea", "menina", "moça", "garota", "dama", "f"}

Isto garante que o jogador nunca fique preso por não saber a palavra exata.

👤 7. PERSONALIZAÇÃO POR PRONOME (VIA PRONOUNS.LUA)
================================================================================

A função NpcUtils.getTratamento deve usar Pronouns.getSubject(player) para
determinar o tratamento correto, em vez de depender de storage 100001.

Exemplo de implementação interna:

lua
function NpcUtils.getTratamento(player)
    local subject = Pronouns.getSubject(player)
    if subject == "she" then
        return { artigo = "a", vocativo = "senhora", sufixoVoc = "a" }
    elseif subject == "he" then
        return { artigo = "o", vocativo = "senhor", sufixoVoc = "o" }
    else
        return { artigo = "", vocativo = "viajante", sufixoVoc = "e" }
    end
end

================================================================================
🧬 9. PERSONALIDADE DO NPC E PROGRESSÃO DE INTIMIDADE
================================================================================

Cada NPC no MCR tem uma personalidade definida. Níveis de intimidade (usando
storages 100010+):
0: Desconhecido
1: Conhecido
2: Amigo
3: Confidente

Exemplos de personalidades:

Sábio caloroso: pergunta o pronome logo no primeiro encontro

Guerreiro rude: usa termos neutros, não pergunta

Místico enigmático: pergunta após intimidade 2

Comerciante astuto: pergunta subtilmente no meio do diálogo

================================================================================
🔑 10. DESTAQUE DE PALAVRAS COM {CHAVES}
================================================================================

O OTClient Redemption reconhece palavras entre {chaves} como destaque visual
(geralmente azul ou sublinhado), herança do protocolo Tibia.

Convenção MCR:

Ações: {aceitar}, {entregar}, {missão}

Itens: {5 Lágrimas de Sereia}

Locais: {Lago dos Reflexos}

Recompensas: {500 pontos de experiência}

⚠️ NUNCA use "}?" junto; sempre "} ?" para evitar erro visual no cliente.
Exemplo correcto: "Você {aceita} ?"
Exemplo INCORRECTO: "Você {aceita}?"

Não use tags {{#RRGGBB}} – NÃO SÃO SUPORTADAS.
