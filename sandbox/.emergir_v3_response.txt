### Sistema de Sabedoria: SessionCache pre-carga + KG + busca hierarquica

**O que significa?**

O Sistema de Sabedoria refere-se a um componente do projeto MCR (Tibia OTServ) que visa melhorar a capacidade de conhecimento e aprendizado do servidor. Este sistema é composto por três elementos principais:

1. **SessionCache pre-carga**: Uma técnica de otimização que carrega antecipadamente informações relevantes para sessões de jogo, reduzindo o tempo necessário para acessar essas informações durante a jogatina.
2. **KG (Knowledge Graph)**: Um gráfico de conhecimento que representa informações estruturadas sobre o mundo do Tibia, incluindo personagens, itens, monstros e locais. O KG permite uma busca hierárquica eficiente dessas informações.
3. **Busca hierárquica**: Uma estratégia para navegar e recuperar informações no Knowledge Graph de forma estruturada e organizada.

**Por que é relevante para o projeto MCR?**

Este sistema é crucial para melhorar a experiência do jogador em vários aspectos:

- **Performance**: Ao pre-carregar informações relevantes, o SessionCache reduz a latência na busca de dados, tornando o jogo mais fluido e responsivo.
- **Conhecimento dinâmico**: O Knowledge Graph permite que o servidor mantenha um conhecimento atualizado do mundo do Tibia, facilitando a implementação de eventos, quests e outros elementos dinâmicos.
- **Aprendizado contextual**: A busca hierárquica no KG ajuda a fornecer informações relevantes e contextuais aos jogadores, melhorando a imersão e o engajamento.

**Qual seu contexto?**

O Sistema de Sabedoria é parte integrante do projeto MCR, que visa criar um servidor customizado para o jogo Tibia. Este sistema é implementado em Python e está sendo desenvolvido em colaboração com outros componentes como o FAST (Decider) e o SPA (Sistema de Progressão do Aventureiro). O objetivo é criar uma experiência de jogo mais rica, dinâmica e interativa.

### Comando executado: master

**O que significa?**

O comando "master" refere-se à execução de um script ou programa chamado "MasterAgent". Este agente é responsável por coordenar várias tarefas relacionadas ao Sistema de Sabedoria, como buscar informações no Knowledge Graph e processá-las.

**Por que é relevante para o projeto MCR?**

O MasterAgent é crucial porque:

- **Coordenação centralizada**: Ele coordena as atividades do Sistema de Sabedoria, garantindo que todas as tarefas sejam executadas corretamente e em ordem.
- **Eficiência operacional**: Ao agilizar o processamento de informações, o MasterAgent contribui para a eficiência geral do servidor, permitindo uma melhor experiência para os jogadores.
- **Monitoramento e feedback**: O resultado da execução do MasterAgent (4/4 subtarefas OK) indica que todas as tarefas foram concluídas com sucesso, fornecendo um feedback valioso sobre o desempenho do sistema.

**Qual seu contexto?**

O comando "master" é executado no ambiente de desenvolvimento do projeto MCR. Ele faz parte da fase de testes e validação do Sistema de Sabedoria, onde os desenvolvedores verificam se todas as funcionalidades estão funcionando conforme o esperado. O tempo de execução (47.2s) é um indicador importante para otimizar ainda mais o desempenho do sistema.

Imagine que cada luz na árvore de Natal é um pequeno portão de conhecimento ou habilidade, e apenas resolvendo o desafio associado a essa luz (ou "pergunta" do Sistema de Sabedoria) a luz se ilumina completamente.

### Analogia: Luz Magica da Sabedoria

Imagine que cada jogador é uma pessoa em uma floresta mágica. Para atravessar essa floresta e encontrar o presente de Natal oculto, eles precisam resolver diversos desafios, como resolver enigmas, enfrentar criaturas fantasmas ou coletar itens escondidos. Cada desafio representa um pouco mais do conhecimento necessário para avançar.

A árvore de Natal na floresta mágica é a fonte principal da energia que sustenta tudo. Quando os jogadores resolverem todos os desafios associados as luzes, elas se iluminam completamente, iluminando o caminho e permitindo que eles encontrem o presente oculto.

### Conexão com MCR

- **Sistema de Sabedoria**: Cada "luz" na árvore de Natal representa uma habilidade ou conhecimento que os jogadores precisam adquirir. O Sistema de Sabedoria do projeto MCR é essencial para fornecer essa informação.
  
- **Decider (FAST)**: Como o Decider, a árvore de Natal pode ser considerada uma fonte "smarter" da energia mágica que impulsiona os jogadores em suas jornadas. Ela decide quais desafios devem aparecer para cada jogador com base no seu nível atual.

- **Sistema de Progressão do Aventureiro (SPA)**: O progresso dos jogadores na árvore de Natal é diretamente relacionado ao progresso deles no jogo. À medida que os jogadores resolvem mais desafios, eles avançam e descobrem luzes novas.

- **Cache inteligente do Knowledge Graph (V12)**: A "luz mágica" também pode ser comparada com o cache V12 do KG. Ela armazena informações e os conhecimentos necessários para iluminar as luzes, garantindo que os jogadores possam resolver os desafios de forma eficiente.

- **Questões Contextuais (SHC)**: O Sistema de Habilidades Contextuais pode ser integrado com a árvore de Natal. Cada luz na árvore poderia apresentar uma questão contextual que varia dependendo do contexto no qual o jogador se encontra.

- **Multiversos Virtuais**: A "floresta mágica" da Arvore de Natal poderia representar um multiverso virtual onde cada luz é uma dimensão diferente, e resolver os desafios permite explorar essas novas realidades.

A integração do Sistema de Sabedoria com a Arvores de Natal festivas cria uma experiência inovadora, onde a jornada para a sabedoria se torna uma encantadora aventura na floresta mágica.
