Aqui está a definição da tabela MySQL para armazenar o progresso dos dominios SPA dos jogadores:
```sql
CREATE TABLE dominio_progresso (
  id INT AUTO_INCREMENT PRIMARY KEY,
  account_id INT NOT NULL,
  player_id INT NOT NULL,
  domain_id INT NOT NULL,
  nivel TINYINT UNSIGNED NOT NULL DEFAULT 1,
  afinidade TINYINT UNSIGNED NOT NULL DEFAULT 0,
  data_atualizacao TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,

  INDEX idx_account_id (account_id),
  INDEX idx_player_id (player_id),
  INDEX idx_domain_id (domain_id),

  FOREIGN KEY (account_id) REFERENCES contas(id),
  FOREIGN KEY (player_id) REFERENCES jogadores(id),
  FOREIGN KEY (domain_id) REFERENCES dominios(id)
);
```
Explicação dos campos:

* `id`: Chave primária da tabela, auto-incremental.
* `account_id`: ID do conta do jogador no sistema de contas.
* `player_id`: ID do jogador no sistema de jogadores.
* `domain_id`: ID do domínio SPA que o jogador está progressando.
* `nivel`: Nível atual do jogador no domínio, com valor inicial 1 e máximo 100 (por exemplo).
* `afinidade`: Afinidade do jogador no domínio, com valor inicial 0 e máximo 100 (por exemplo).
* `data_atualizacao`: Data e hora da última atualização do progresso do jogador.

Os índices são criados para melhorar a performance das consultas que buscam por conta_id, player_id ou domain_id. As chaves estrangeiras garantem que os dados sejam consistentes entre as tabelas de contas, jogadores e dominios.

Lembre-se de criar as tabelas de contas, jogadores e dominios antes de criar a tabela de progresso dos dominios SPA.
```sql
CREATE TABLE contas (
  id INT AUTO_INCREMENT PRIMARY KEY,
  nome VARCHAR(255) NOT NULL,
  email VARCHAR(255) NOT NULL,
  senha VARCHAR(255) NOT NULL
);

CREATE TABLE jogadores (
  id INT AUTO_INCREMENT PRIMARY KEY,
  nome VARCHAR(255) NOT NULL,
  conta_id INT NOT NULL,
  FOREIGN KEY (conta_id) REFERENCES contas(id)
);

CREATE TABLE dominios (
  id INT AUTO_INCREMENT PRIMARY KEY,
  nome VARCHAR(255) NOT NULL
);
```
