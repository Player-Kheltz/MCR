## Bug na linha 74 do cmd_fix_excepts.py

### Identificacao do bug
Linha 74: `rel = os.path.relpath(fpath, os.path.dirname(os.path.dirname(__file__)) + '/..')`

O bug esta no calculo do diretorio BASE usado para o caminho relativo.

### Explicacao
`__file__` em `cmd_fix_excepts.py` vale: `E:\Projeto MCR\scripts\mcr_devia\comandos\cmd_fix_excepts.py`
- `os.path.dirname(__file__)` = `E:\Projeto MCR\scripts\mcr_devia\comandos`
- `os.path.dirname(os.path.dirname(__file__))` = `E:\Projeto MCR\scripts\mcr_devia`
- Depois concatena `+ '/..'`: `E:\Projeto MCR\scripts\mcr_devia/..` = `E:\Projeto MCR\scripts`

Entao `rel = os.path.relpath(fpath, 'E:\Projeto MCR\scripts')` - o caminho relativo sera calculado considerando `scripts/` como base.

**O problema:** O projeto base e `E:\Projeto MCR`. O programador provavelmente queria calcular o caminho relativo a RAIZ do projeto, nao a `scripts/`. Para isso, precisaria de TRES `os.path.dirname()` (subindo de comandos/ -> mcr_devia/ -> scripts/ -> raiz), nao dois.

### Codigo corrigido
```python
rel = os.path.relpath(fpath, os.path.dirname(os.path.dirname(os.path.dirname(__file__))))
```

Ou, mais legivel:
```python
BASE_PROJETO = os.path.abspath(os.path.join(os.path.dirname(__file__), '..', '..', '..'))
rel = os.path.relpath(fpath, BASE_PROJETO)
```

### Impacto
Com o bug, a saida mostra caminhos relativos incorretos como `..\..\foo.py` em vez de `scripts\mcr_devia\comandos\foo.py`, dificultando para o usuario identificar qual arquivo foi modificado.
