## Supervisor, Context Provider e Template Engine: Diferencas e Relacoes

### (1) Responsabilidade de cada um

**Supervisor (orquestracao)**
- Responsabilidade: coordenar o fluxo de execucao, decidir QUAL acao tomar baseado na entrada
- Gerencia estado da conversa, decide quando chamar ferramentas, quando perguntar, quando responder
- Age como um "controller" no padrao MVC - nao executa trabalho diretamente, mas delega
- Ex: Supervisor V12 decide: "essa pergunta e factual? Entao vou para o KG. E criativa? Vou para o orquestrador de lore."

**Context Provider (injecao de contexto)**
- Responsabilidade: buscar, agregar e fornecer informacao relevante para a tarefa atual
- Consulta multiplas fontes (banco de dados, arquivos, API, memoria) e retorna apenas o que e relevante
- Nao decide o que fazer com o contexto - apenas fornece
- Ex: ContextCrew busca em paralelo no KG, docs, web, codigo fonte e retorna trechos relevantes

**Template Engine (geracao de saida)**
- Responsabilidade: gerar saida estruturada a partir de um template fixo + dados injetados
- Garante consistencia de formato (sempre produz a mesma estrutura)
- Separa a apresentacao (template) da logica (dados)
- Ex: Orquestrador que pega um template de lore, injeta contexto e dados, e gera o prompt final

### (2) Como se diferenciam

| Aspecto | Supervisor | Context Provider | Template Engine |
|---------|-----------|-----------------|-----------------|
| Pergunta | "O QUE fazer?" | "COM O QUE fazer?" | "COMO apresentar?" |
| Entrada | Requisicao do usuario | Consulta (termos, ID) | Dados + Template |
| Saida | Decisao/acao | Dados brutos | Texto formatado |
| Estado | Mantem estado da sessao | Cache de consultas | Stateless |
| Falha | Fallback para outro fluxo | Retorna vazio ou parcial | Usa template sem dados |

### (3) Exemplo de fluxo conjunto

**Cenario:** Usuario pergunta "Explique o que e alucinacao em IAs e como o projeto lida com isso"

1. **Supervisor** recebe a pergunta, analisa a intencao ("pergunta tecnica sobre o projeto"), decide que precisa de contexto do KG + docs. Aciona o Context Provider com os termos "alucinacao, IA, projeto".

2. **Context Provider** busca em paralelo:
   - KG: encontra lessons sobre alucinacao
   - Docs: encontra secoes em lessons/recentes.md
   - Codigo: encontra validacao semÃ¢ntica no _teste_cego.py
   Agrega tudo e retorna um bloco unico de contexto relevante.

3. **Supervisor** recebe o contexto e decide qual template usar. Identifica que e uma pergunta explicativa, nao tecnica, entao escolhe o template "perguntar" do Template Engine.

4. **Template Engine** pega o template "perguntar", injeta o contexto do Provider + a pergunta original, e gera o prompt final para o LLM.

5. **Supervisor** envia o prompt para o LLM, recebe a resposta, e retorna ao usuario.

**Fluxo em codigo conceitual:**
```python
def processar(pergunta):
    intencao = supervisor.classificar(pergunta)  # "explicativa"
    if intencao == "explicativa":
        termos = extrair_termos(pergunta)  # ["alucinacao", "IA"]
        contexto = context_provider.buscar(termos)  # busca KG + docs + codigo
        template = template_engine.selecionar("perguntar")
        prompt = template.preencher(pergunta=pergunta, contexto=contexto)
        resposta = llm.gerar(prompt)
        return resposta
```
