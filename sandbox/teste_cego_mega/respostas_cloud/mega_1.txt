## DataLake — Diagnóstico e Solução Completa

### Análise do Código

O `datalake.py` contém duas classes com responsabilidades sobrepostas. A classe `DataLake` é um pipeline de processamento que tenta fazer tudo sozinha: receber streams, transformar itens, aplicar filtros, manter cache global, salvar arquivos e gerar estatísticas. A classe `StreamSimulator` gera dados de sensores fictícios mas tem um problema conceitual: cada chamada de `gerar_leitura()` adiciona ao `self.leituras` sem limite, outro memory leak em potencial.

A classe `DataLake` usa uma variável de classe `_cache_global = {}` que é COMPARTILHADA entre todas as instâncias. Isso significa que processar dados em uma instância afeta o cache de todas as outras — um bug sutil de isolamento.

### Bugs Encontrados

**BUG 1 – Memory leak no histórico (linha 25 + linha 38)**: `self.historico.append()` acumula cada stream processado sem nunca limpar. Após N iterações, a lista contém N entradas, cada uma com timestamp, dados completos e contador. Com 10 mil iterações (~3.6MB), o sistema começa a degradar. Com 1 milhão (~360MB), trava. **Correção**: usar `collections.deque(maxlen=10000)` ou implementar política de retenção temporal.

```python
from collections import deque
# No __init__:
self.historico = deque(maxlen=10000)
```

**BUG 2 – Race condition no contador (linha 35)**: `self._contador = self._contador + 1` não é uma operação atômica. Em Python, isso é: (1) ler, (2) somar, (3) escrever. Entre a leitura e a escrita, outra thread pode ler o mesmo valor, resultando em dois incrementos com o mesmo resultado. Em produção com streams concorrentes, contagens são perdidas. **Correção**:

```python
with self._lock:
    self._contador += 1
```

**BUG 3 – Security hole com eval() (linha 77)**: `eval(filtro)` executa código Python arbitrário recebido nos dados do stream. Um invasor que controle os dados do sensor pode enviar algo como `{"filtro": "__import__('os').system('rm -rf /')"}` e executar comandos no servidor. Isso é CRÍTICO e deve ser corrigido URGENTE. **Correção**: substituir eval por um interpretador de filtros seguro:

```python
def _aplicar_filtro(self, item):
    filtro = item.get('filtro', '')
    if not filtro or 'filtro' not in item:
        return
    # Interpretador seguro baseado em operadores
    ops = {'>': lambda a, b: a > b, '<': lambda a, b: a < b, '==': lambda a, b: a == b,
           '>=': lambda a, b: a >= b, '<=': lambda a, b: a <= b}
    for campo, op_str, valor_str in self._parse_filtro(filtro):
        op = ops.get(op_str)
        if op and campo in item:
            item['filtrado'] = op(item[campo], float(valor_str))
```

**BUG 4 – Cache sem limite (linha 60)**: O `_cache_global` é um dicionário de classe que cresce indefinidamente. Cada item único jamais sai do cache. Combinado com o BUG 7 (variável de classe), isso significa que TODAS as instâncias contribuem para UM ÚNICO cache que nunca é limpo. **Correção**: implementar LRU com `OrderedDict` e limite máximo.

**BUG 5 – Divisão por zero (linha 68)**: `len(valores_positivos)` pode ser 0 se todos os valores de `item['valores']` forem zero ou negativos. `sum(item['valores']) / 0` levanta `ZeroDivisionError`. **Correção**:

```python
if valores_positivos:
    media = sum(valores_positivos) / len(valores_positivos)
else:
    media = 0.0
```

**BUG 6 – Path traversal (linha 85)**: `os.path.join("./data", nome_arquivo)` não valida o nome. Alguém pode passar `../../etc/passwd` como nome, fazendo o sistema escrever fora do diretório permitido. **Correção**:

```python
import re
if not re.match(r'^[\w\-\.]+$', nome_arquivo):
    raise ValueError(f"Nome de arquivo invalido: {nome_arquivo}")
caminho = os.path.join("./data", nome_arquivo)
```

**BUG 7 – Cache global compartilhado (linha 20)**: `_cache_global = {}` é um dicionário no nível da CLASSE, não da instância. Todas as instâncias de `DataLake` compartilham o mesmo cache. Se dois streams diferentes estiverem sendo processados por instâncias diferentes, eles interferem um no outro. **Correção**: mover para `self._cache = {}` no `__init__`.

**BUG 8 – Efeito colateral (linha 63)**: `item['transformado'] = True` modifica o ITEM ORIGINAL, não a cópia armazenada no cache. Isso corrompe os dados de entrada. **Correção**: modificar a cópia, não o original.

**BUG 9 – Loop infinito (linha 95)**: `while i < tamanho:`... mas `i` nunca é incrementado! O loop roda para sempre com `i=0`. **Correção**: adicionar `i += 1` no final do while.

**BUG 10 – Arquivo não fechado (linha 88)**: `json.dump(self.historico, f)` escreve no arquivo mas `f.close()` nunca é chamado. Em sistemas Unix, isso pode causar perda de dados se o buffer não for descarregado. **Correção**: usar `with open(caminho, 'w') as f:`.

### Módulo Validador de Streams

```python
"""validador_stream.py - Validacao de streams de dados industriais."""
from typing import List, Tuple, Dict, Any

class ErroStream:
    def __init__(self, stream_id: str, campo: str, valor: float, regra: str):
        self.stream_id = stream_id
        self.campo = campo
        self.valor = valor
        self.regra = regra
    
    def __repr__(self):
        return f"[{self.stream_id}] {self.campo}={self.valor} viola {self.regra}"

class ValidadorStream:
    REGRAS = {
        'temp': (0, 100, 'temperatura fora de 0-100'),
        'pressao': (0.5, 30, 'pressao fora de 0.5-30'),
        'vibracao': (0, 60, 'vibracao fora de 0-60'),
        'pressure': (0.5, 30, 'pressure fora de 0.5-30'),
        'vibration': (0, 60, 'vibration fora de 0-60'),
    }
    
    def __init__(self):
        self.fila_rejeicao: List[Dict] = []
        self.total_validados = 0
    
    def validar(self, stream: Dict[str, Any]) -> Tuple[bool, List[ErroStream], List[str]]:
        erros = []
        alertas = []
        
        for campo, valor in stream.items():
            if campo in ('timestamp', 'valores', 'stream', 'filtro'):
                continue
            if isinstance(valor, (int, float)):
                for prefixo, (min_v, max_v, msg) in self.REGRAS.items():
                    if campo.lower().startswith(prefixo):
                        if not (min_v <= valor <= max_v):
                            erros.append(ErroStream(stream.get('stream', '?'), campo, valor, msg))
                        break
        
        self.total_validados += 1
        
        if erros:
            self.fila_rejeicao.append({
                'stream': stream,
                'erros': [str(e) for e in erros],
                'timestamp': __import__('time').time()
            })
        
        return len(erros) == 0, erros, alertas
    
    def relatorio(self) -> Dict:
        return {
            'total_validados': self.total_validados,
            'fila_rejeicao': len(self.fila_rejeicao),
        }
```

### Nova Arquitetura em 3 Camadas

```
[INGESTAO]               [PROCESSAMENTO]           [ARMAZENAMENTO]
StreamCollector     →    DataPipeline         →    DataStore
    |                       |                       |
    v                       v                       v
StreamValidator      StreamTransformer       FileRepository
                     MetricsEngine           DatabaseRepository
                     FilterEngine
                     
Fluxo:
1. StreamCollector.receber(stream_id, dados) → StreamValidator.validar(dados)
2. Se valido: DataPipeline.executar(dados) → StreamTransformer + MetricsEngine
3. DataStore.persistir(resultado) → FileRepository ou DatabaseRepository
4. Se invalido: StreamCollector.rejeitar(stream_id, erros)
```

Cada classe tem UMA responsabilidade. O `DataPipeline` orquestra o fluxo mas não implementa validação, transformação ou armazenamento.

### Revisão Priorizada

1. **CRÍTICO – eval() linha 77**: execução remota de código. Corrigir IMEDIATAMENTE.
2. **CRÍTICO – Path traversal linha 85**: escrita arbitrária de arquivos.
3. **ALTO – Cache global linha 20**: dados vazam entre instâncias.
4. **ALTO – Loop infinito linha 95**: trava o sistema se chamado.
5. **ALTO – Memory leak linhas 25+38+60**: causa crash após horas de uso.
6. **MÉDIO – Race condition linha 35**: dados inconsistentes em concorrência.
7. **MÉDIO – Arquivo não fechado linha 89**: perda de dados.
8. **MÉDIO – Efeito colateral linha 63**: corrompe dados de entrada.
9. **BAIXO – StreamSimulator.leituras sem limite**: outro leak menor.

### A Lenda do DataLake

Dizem que antes dos servidores existir, os dados fluíam livres como água em um lago mágico chamado DataLake, guardado pelo espírito digital YARA, que bebia streams de informação e os transformava em conhecimento cristalino. Mas um dia, um engenheiro chamado CRIUS tentou roubar o lago inteiro sem filtrar as impurezas. Ele construiu um pipeline que sugava tudo — dados puros e lixo juntos — e o lago começou a transbordar, criando o primeiro Memory Leak da história. YARA, enfurecida, amaldiçoou todo pipeline que não tivesse validação: "todo dado que entra sem filtro um dia transborda sem controle". Desde então, engenheiros de dados honram YARA construindo válvulas de escape (deques com limite), filtros de pureza (validadores) e diques de contenção (tratamento de erros). O nome DataLake é um lembrete: dados parados viram pântano; dados filtrados viram conhecimento.

### Diagnóstico de Causa Raiz

**Causa raiz do crash**: A combinação dos BUGS 1, 4 e 7 cria uma degradação exponencial de memória.

**Mecanismo**:
1. A cada chamada de `processar_stream()`, um novo dicionário é adicionado a `self.historico` (BUG 1) e uma entrada é adicionada a `_cache_global` (BUG 4). Ambos NUNCA são removidos.
2. Como `_cache_global` é uma variável de CLASSE (BUG 7), MÚLTIPLAS instâncias alimentam o mesmo cache.
3. Cada entrada consome ~500 bytes. Após 1 hora com 100 streams/segundo: 100 × 3600 × 500 = 180MB. Após 6 horas: 1GB+.
4. O `time.sleep(0.1)` inexistente no código atual significa que não há pausa — a degradação ocorre o mais rápido possível.

**Reprodução**: `p = DataLake(); [p.processar_stream(f"stream_{i}", [{"id": i}]) for i in range(100000)]` e monitorar a memória com `tracemalloc` ou `psutil`.

**Confirmação**: `p.estatisticas()['cache_size']` cresce monotonicamente. `len(p.historico)` idem.

**Soluções**:
1. Imediato: deque(maxlen=10000) para historico
2. Imediato: LRU cache com maxsize=1000 para _cache
3. Curto prazo: mover _cache para instância (self._cache)
4. Médio prazo: TTL no cache (expirar entradas após N segundos)

### Refatoração do processar_stream

```python
def processar_stream(self, stream_id, dados):
    """Processa stream em 5 etapas isoladas."""
    self._executando = True
    try:
        itens_validados = self._validar_dados(dados)
        itens_transformados = self._transformar_lote(itens_validados)
        self._aplicar_filtros_lote(itens_transformados)
        self._incrementar_contador()
        self._registrar_historico(stream_id, itens_transformados)
        return self._contador
    finally:
        self._executando = False

def _validar_dados(self, dados):
    """Etapa 1: validacao basica."""
    return [d for d in dados if isinstance(d, dict)]

def _transformar_lote(self, dados):
    """Etapa 2: transformacao em lote."""
    return [self._transformar_item(item) for item in dados]

def _transformar_item(self, item):
    """Transforma UM item com cache seguro."""
    chave = json.dumps(item, sort_keys=True)
    if chave not in self._cache:
        self._cache[chave] = item.copy()
        self._cache[chave]['transformado'] = True
        self._limpar_cache()
    return self._cache[chave].copy()

def _aplicar_filtros_lote(self, dados):
    """Etapa 3: filtros seguros."""
    for item in dados:
        self._aplicar_filtro_seguro(item)

def _aplicar_filtro_seguro(self, item):
    """Filtro SEM eval. Usa operadores seguros."""
    filtro = item.get('filtro', '')
    if filtro.startswith('valor '):
        partes = filtro[6:].split()
        if len(partes) == 2 and partes[0] in ('>', '<', '>=', '<=', '=='):
            campo, op, val = 'valor', partes[0], float(partes[1])
            ops = {'>': lambda a, b: a > b, '<': lambda a, b: a < b}
            if op in ops:
                item['filtrado'] = ops[op](item.get(campo, 0), val)

def _incrementar_contador(self):
    """Etapa 4: contador com lock."""
    with self._lock:
        self._contador += 1

def _registrar_historico(self, stream_id, dados):
    """Etapa 5: registro com limite."""
    self.historico.append({
        'stream_id': stream_id, 'dados': dados,
        'timestamp': time.time(), 'contador': self._contador
    })
    if len(self.historico) > 10000:
        for _ in range(len(self.historico) - 10000):
            self.historico.popleft()

def _limpar_cache(self):
    """Mantem cache com limite LRU."""
    if len(self._cache) > 1000:
        sobra = len(self._cache) - 1000
        for _ in range(sobra):
            self._cache.pop(next(iter(self._cache)))
```

### Lições para o Novo Desenvolvedor

Três lições que este código ensina: **primeiro**, `eval()` em dados externos não é um bug — é uma vulnerabilidade que entrega o servidor para um atacante. Nunca, em hipótese alguma, use `eval()` ou `exec()` com entrada do usuário. **Segundo**, toda estrutura de dados que cresce com append e sem limite é uma bomba-relógio de memória — use sempre `deque(maxlen=...)`, `LRU cache` ou políticas de retenção desde o início, não como depois. **Terceiro**, uma variável de classe `{}` parece inofensiva até que duas instâncias compartilhem o mesmo cache e comecem a corromper dados uma da outra — estado global compartilhado é a causa mais sutil de bugs em produção e deve ser evitado a todo custo. O DataLake não quebrou por acaso: quebrou porque cada bug individual parecia "pequeno demais para se preocupar" — até que todos eles se somaram.
