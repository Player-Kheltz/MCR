## Análise da Configuração de C++ Runtime em main.cpp

### Propósito da Configuração (L27-35)

O código em `main.cpp:L27-35` implementa uma **localidade híbrida** para o C++ Runtime. O propósito é duplo:

1. **Base "C"** (linha 29): Garante que formatação numérica use o padrão POSIX (ponto como separador decimal, sem separador de milhares). Isso protege o MariaDB SQL de receber números formatados no padrão brasileiro (ex: "1.234,56" em vez de "1234.56").

2. **Classificação ".1252"** (linha 30): Injeta apenas o `ctype` (classificação de caracteres) do locale Windows-1252, permitindo que funções C como `tolower()` e `isalpha()` funcionem corretamente com acentos (Latin-1), sem afetar a formatação numérica.

### Riscos de Segurança

1. **Catch genérico (L32-34)**: `catch (...)` captura QUALQUER exceção, incluindo falhas graves de sistema. Se `std::locale("C")` também falhar (improvável, mas possível em ambientes com locale corrompido), o programa continua com locale indefinido. Isso pode causar comportamento silenciosamente incorreto em operações de string.

2. **Locale global mutável (L31)**: `std::locale::global()` altera o locale de TODO o processo. Se o servidor usar bibliotecas terceiras que dependem de um locale específico (ex: bibliotecas de formatação que esperam "en_US.UTF-8"), essa mudança global pode quebrá-las silenciosamente.

3. **Dependência de encoding Windows (L30)**: O código assume que ".1252" está disponível no sistema. Em sistemas Linux ou Mac, esse locale pode não existir, fazendo o `try` capturar a exceção e cair no fallback "C" — que não tem suporte a acentos. O comportamento difere por plataforma sem aviso.

### Proposta em Rust

```rust
/// Configuracao de locale segura e explicita para o servidor.
/// Rust nao depende de locale global, entao a abordagem e diferente.

use std::env;

pub struct RuntimeConfig {
    /// Formato numerico para SQL (sempre ponto como separador)
    pub sql_numeric_format: NumericFormat,
    /// Codificacao de caracteres para saida
    pub output_encoding: Encoding,
}

pub enum NumericFormat {
    /// Ponto como separador decimal, sem separador de milhares (seguro para SQL)
    Posix,
}

pub enum Encoding {
    Utf8,
    Windows1252,
}

impl RuntimeConfig {
    /// Inicializa configuracao sem locale global (seguro para threads)
    pub fn new() -> Self {
        // Em Rust, nao precisamos mudar locale global.
        // Cada componente define seu formato explicitamente.
        Self {
            sql_numeric_format: NumericFormat::Posix,
            output_encoding: if cfg!(windows) {
                Encoding::Windows1252
            } else {
                Encoding::Utf8
            },
        }
    }
    
    /// Formata numero para SQL (sempre com ponto, seguro contra SQL injection numerico)
    pub fn format_for_sql(&self, valor: f64) -> String {
        format!("{:.6}", valor) // Sempre ponto, 6 casas decimais
    }
    
    /// Converte string para encoding de saida
    pub fn encode_output(&self, texto: &str) -> Vec<u8> {
        match self.output_encoding {
            Encoding::Utf8 => texto.as_bytes().to_vec(),
            Encoding::Windows1252 => {
                // Conversao simplificada de UTF-8 para Windows-1252
                texto.chars().map(|c| {
                    match c as u32 {
                        0x00..=0x7F => c as u8,
                        0xC0..=0xFF => (c as u8) & 0x7F, // Acentos Latin-1
                        _ => b'?', // Caractere nao mapeado
                    }
                }).collect()
            }
        }
    }
}

#[cfg(test)]
mod tests {
    use super::*;
    
    #[test]
    fn test_sql_format_usar_ponto() {
        let config = RuntimeConfig::new();
        assert_eq!(config.format_for_sql(1234.56), "1234.560000");
    }
    
    #[test]
    fn test_encoding_windows1252() {
        let mut config = RuntimeConfig::new();
        config.output_encoding = Encoding::Windows1252;
        let result = config.encode_output("coração");
        assert!(result.len() > 0);
    }
}
```

### Benefícios da abordagem Rust

| Problema C++ | Solução Rust |
|-------------|-------------|
| Locale global afeta todo o processo | Configuração por componente, sem estado global |
| `catch (...)` esconde erros | `Result<T, E>` explícito, sem catch genérico |
| Comportamento diferente por plataforma sem aviso | `cfg!(windows)` explícito, documentado |
| Dependência de locale ".1252" pode falhar | Encoding explícito por enum, sem dependência externa |
| Formatação numérica insegura para SQL | `format_for_sql()` sempre produz formato seguro |
