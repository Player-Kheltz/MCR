## Análise e Refatoração do FileProcessor

### 1. Problemas Identificados

**Performance:**
- Linha 16: `processar_tudo()` itera sequencialmente. Cada arquivo espera o anterior terminar, incluindo chamadas de rede em `_verificar_servidor` (linha 52) que são síncronas e bloqueantes.
- Linha 52: `requests.get()` é síncrono — bloqueia a thread enquanto espera resposta HTTP. Com N arquivos, o tempo total = soma dos tempos individuais, não o máximo.

**Manutenção:**
- Linha 21-30: Validação de dados misturada com processamento. Viola Single Responsibility Principle.
- Linha 10: `resultados` e `erros` são atributos de instância mutáveis — causa efeitos colaterais se o mesmo processador for reusado.
- Linha 47: Tratamento de exceção genérico `except Exception` — esconde erros inesperados.

**Segurança:**
- Linha 52: `requests.get()` sem verificação de certificado SSL (verify=True é o padrão, mas não explícito).
- Linha 61: `json.dump` sem gerenciador de contexto explícito (já usa `with`, ok).

### 2. Código Refatorado com Async/await

```python
"""
processador_async.py - Processamento paralelo de arquivos com asyncio.

Uso:
    processor = AsyncFileProcessor(timeout=5.0, max_workers=10)
    resultados = asyncio.run(processor.processar_tudo(["config1.json", "config2.json"]))
"""
import asyncio
import aiohttp
import aiofiles
import json
import logging
from typing import List, Dict, Optional, Tuple
from dataclasses import dataclass, field
from enum import Enum

# Logging estruturado
logging.basicConfig(level=logging.INFO, format='%(asctime)s | %(levelname)s | %(message)s')
logger = logging.getLogger(__name__)


class StatusServidor(Enum):
    ONLINE = "online"
    OFFLINE = "offline"
    ERRO = "erro"


@dataclass
class ResultadoProcessamento:
    arquivo: str
    servidor: Optional[str] = None
    porta: Optional[int] = None
    status: StatusServidor = StatusServidor.ERRO
    tamanho: int = 0
    erro: Optional[str] = None


class ValidadorConfig:
    """Responsável apenas por validar configurações."""
    
    @staticmethod
    def validar(arquivo: str, dados: dict) -> Optional[str]:
        """Valida dados de configuracao. Retorna None se valido, ou string de erro."""
        if not isinstance(dados, dict):
            return "formato invalido"
        if 'servidor' not in dados:
            return "sem campo servidor"
        if 'porta' not in dados:
            return "sem campo porta"
        if not isinstance(dados['porta'], (int, float)):
            return "porta deve ser numero"
        if not 1 <= dados['porta'] <= 65535:
            return "porta fora do intervalo 1-65535"
        return None


class VerificadorServidor:
    """Responsável por verificar disponibilidade de servidores com retry."""
    
    def __init__(self, timeout: float = 5.0, max_retries: int = 3, base_delay: float = 0.5):
        self.timeout = timeout
        self.max_retries = max_retries
        self.base_delay = base_delay
    
    async def verificar(self, session: aiohttp.ClientSession, servidor: str, porta: int) -> StatusServidor:
        """Verifica servidor com retry exponencial."""
        url = f"http://{servidor}:{porta}/health"
        
        for tentativa in range(self.max_retries):
            try:
                async with session.get(url, timeout=aiohttp.ClientTimeout(total=self.timeout)) as resp:
                    if resp.status == 200:
                        return StatusServidor.ONLINE
                    else:
                        return StatusServidor.OFFLINE
            except (asyncio.TimeoutError, aiohttp.ClientError) as e:
                if tentativa < self.max_retries - 1:
                    delay = self.base_delay * (2 ** tentativa)  # Backoff exponencial
                    logger.warning(f"Tentativa {tentativa + 1} falhou para {url}: {e}. Retry em {delay:.1f}s")
                    await asyncio.sleep(delay)
                else:
                    logger.error(f"Todas as {self.max_retries} tentativas falharam para {url}: {e}")
                    return StatusServidor.OFFLINE
        return StatusServidor.OFFLINE


class AsyncFileProcessor:
    """Processador de arquivos assincrono com validacao e verificacao paralela."""
    
    def __init__(self, timeout: float = 5.0, max_retries: int = 3, base_delay: float = 0.5):
        self.timeout = timeout
        self.max_retries = max_retries
        self.base_delay = base_delay
        self.validador = ValidadorConfig()
        self.verificador = VerificadorServidor(timeout, max_retries, base_delay)
    
    async def processar_tudo(self, arquivos: List[str]) -> Tuple[List[ResultadoProcessamento], List[str]]:
        """Processa todos os arquivos em paralelo usando asyncio.gather."""
        async with aiohttp.ClientSession() as session:
            tarefas = [self._processar_um(session, arq) for arq in arquivos]
            resultados = await asyncio.gather(*tarefas, return_exceptions=True)
        
        processados = []
        erros = []
        for r in resultados:
            if isinstance(r, Exception):
                erros.append(str(r))
            elif isinstance(r, ResultadoProcessamento):
                if r.erro:
                    erros.append(f"{r.arquivo}: {r.erro}")
                processados.append(r)
        
        return processados, erros
    
    async def _processar_um(self, session: aiohttp.ClientSession, arquivo: str) -> ResultadoProcessamento:
        """Processa UM arquivo: le, valida, verifica servidor."""
        logger.info(f"Processando: {arquivo}")
        
        try:
            # Leitura do arquivo
            async with aiofiles.open(arquivo, 'r') as f:
                conteudo = await f.read()
            
            dados = json.loads(conteudo)
            
            # Validacao (separada do processamento)
            erro_validacao = self.validador.validar(arquivo, dados)
            if erro_validacao:
                return ResultadoProcessamento(arquivo=arquivo, erro=erro_validacao)
            
            # Verificacao do servidor (assincrona, paralela)
            status = await self.verificador.verificar(session, dados['servidor'], dados['porta'])
            
            return ResultadoProcessamento(
                arquivo=arquivo,
                servidor=dados['servidor'],
                porta=dados['porta'],
                status=status,
                tamanho=len(conteudo),
            )
            
        except FileNotFoundError:
            return ResultadoProcessamento(arquivo=arquivo, erro="arquivo nao encontrado")
        except json.JSONDecodeError as e:
            return ResultadoProcessamento(arquivo=arquivo, erro=f"json invalido: {e}")
        except Exception as e:
            logger.exception(f"Erro inesperado em {arquivo}")
            return ResultadoProcessamento(arquivo=arquivo, erro=f"erro inesperado: {e}")


async def main():
    """Exemplo de uso."""
    processor = AsyncFileProcessor(timeout=5.0, max_retries=3)
    arquivos = ["servidor_a.json", "servidor_b.json", "servidor_c.json"]
    
    resultados, erros = await processor.processar_tudo(arquivos)
    
    logger.info(f"Processados: {len(resultados)}, Erros: {len(erros)}")
    for r in resultados:
        logger.info(f"  {r.arquivo}: {r.servidor}:{r.porta} = {r.status.value}")
    for e in erros:
        logger.warning(f"  ERRO: {e}")


if __name__ == "__main__":
    asyncio.run(main())
```

### 3. Melhorias Implementadas

| Problema Original | Solução | Benefício |
|------------------|---------|-----------|
| Sequencial (linha 16) | `asyncio.gather()` com `aiohttp` | Processamento paralelo: N arquivos no tempo do mais lento |
| Sem retry (linha 52) | Backoff exponencial com 3 tentativas | Resiliência a falhas de rede transitórias |
| Timeout fixo (linha 52) | `ClientTimeout` configurável | Controle fino por requisição |
| Validação misturada (linha 21-30) | `ValidadorConfig` separado | SRP, testável isoladamente |
| Except genérico (linha 47) | Exceções específicas (`FileNotFoundError`, `JSONDecodeError`) | Debug mais preciso |
| Sem logging | `logging` estruturado com níveis | Monitoramento em produção |
| Sem type hints | Type hints completos + dataclasses | IDE support, documentação viva |
| Estado mutável (`self.resultados`) | Imutabilidade: cada chamada retorna novos resultados | Sem efeitos colaterais |
| `requests` síncrono | `aiohttp` assíncrono | Non-blocking I/O |

### 4. Boas Práticas Aplicadas

- **S**ingle Responsibility: `ValidadorConfig` valida, `VerificadorServidor` verifica, `AsyncFileProcessor` orquestra
- **O**pen/Closed: novo tipo de validação = novo método no validador, sem mexer no processador
- **L**iskov: `ResultadoProcessamento` é imutável (dataclass frozen)
- **I**nterface Segregation: cada classe tem interface mínima
- **D**ependency Inversion: `VerificadorServidor` recebe timeout/retries por construtor
